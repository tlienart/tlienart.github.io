CV Ridge (part 2)

Where in the first part, we showed that the LOOCV trick could be obtained by using the Sherman-Morrison formula for the inversion of a rank-1 perturbation of an invertible matrix, in the general case where we want to sequentially drop more than one point. It is therefore not a rank-1 perturbation anymore but a sum of rank-1 perturbations.

In these notes we investigate using Sherman-Morrison recursively and discuss in which circumstances this makes sense.

Complexity of the recursion

Let M=∣S∣M=|S|M=∣S∣ the number of rows dropped per selection/fold. Then there are MMM terms in the sum (6). Each bib_ibi​ is given by the application of a p×pp\times pp×p matrix on a vector. So, inherently, the complexity of this recursive inverse is O(Mp2)O(Mp^2)O(Mp2) and it shouldn't be used if MMM is of order comparable to ppp.

In K-folds cross validation, M=O(n/K)M = O(n/K)M=O(n/K) where KKK might typically be around 555 or 101010 so that MMM may very well be of order ppp or larger. We will come back to this in the next section.

Applying RSM in the tall case

One unfortunate element of RSM is that it is not possible (to the best of my knowledge) to easily get the inverse of the matrix for several λλλ without recomputing the recursion each time. This is because the computation of each bib_ibi​ after the initial one will involve non-diagonal matrices that are shaped by λ\lambdaλ. Note that in the LOO case, this was not the case because we only had one diagonal matrix to update which is O(p)O(p)O(p) (i.e. negligible).

The observation above means that for a single selection/fold, we have to pay O(κMp2)O(κMp^2)O(κMp2) if κκκ is the number of λλλ we want to test. Assuming we do KKK folds/selections, the overall cost is therefore O(κKMp2)O(κKMp^2)O(κKMp2) to form all of the eSe_SeS​ vectors.

K-folds CV

In K-folds CV, MMM is O(n/K)O(n/K)O(n/K) and the total cost with RSM is thus O(κnp2)O(κnp^2)O(κnp2).

If, instead, we were to re-compute an SVD of XStXSX_S^tX_SXSt​XS​ per fold and recycle the computations for the λλλ, the cost per fold is O((n−M+κ)p2)O((n - M + κ)p^2)O((n−M+κ)p2) or O(Knp2)O(Knp^2)O(Knp2) overall assuming that κκκ is dominated by nnn.

Since, in general, we are much more likely to want to test many λλλ (large κκκ) with a few folds (with KKK of order 555-101010), doing re-computations per fold makes more sense than applying RSM.

Note: it is still important to recycle computations otherwise we pay O(κKnp2)O(κKnp^2)O(κKnp2) by doing a naive grid-computations for every fold and every λ\lambdaλ; surprisingly this seems to be what is currently done in SkLearn as per these lines of code.

Fat case

Since the complexity of RSM is O(Mp2)O(Mp^2)O(Mp2), it shouldn't be considered. Computing KKK problem with n−Mn-Mn−M rows and recycling computations for the λλλ is best.

Conclusion

So disappointingly the generalisation of the LOO trick to the more general case is not particularly useful compared to simply re-computing things per-folds and recycling computations for the λλλ. This could maybe have been expected, we have to deal with an expansion in MMM terms where MMM scales like nnn in KKK folds with low KKK (the usual case).
