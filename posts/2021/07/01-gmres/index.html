GMRES & Conjugate Gradient (part I)

code, linear algebra

Let A∈Rn×nA \in \mathbb R^{n\times n}A∈Rn×n non singular and b∈Rnb\in \mathbb R^nb∈Rn, we want to find (or approximate) x∈Rnx\in \mathbb R^nx∈Rn such that Ax=bAx = bAx=b. In this first part we briefly discuss direct methods, where one recovers xxx exactly, and a set of simple iterative methods including the Generalised Minimal Residual Method or GMRES, where one constructs a sequence xk→xx_k \to xxk​→x. Most of what we cover also holds for the case where either AAA or bbb have complex entries though we stick to the real case for ease of presentation.

In the second part we discuss how GMRES is related to Krylov Subspaces as well as improvements when it is known that AAA is positive definite or symmetric or both.

Direct methods

There are plenty of direct methods to solve a linear system, a common one is the LU-factorisation where AAA is decomposed into A=LUA=LUA=LU with LLL a lower-triangular matrix and UUU an upper-triangular one.
We won't cover LU here but will briefly cover another approach that leverages the QR-factorisation of AAA which we introduced in a previous post. If A=QRA=QRA=QR with QtQ=QQt=IQ^tQ=QQ^t=IQtQ=QQt=I and RRR upper-triangular, then the problem to solve becomes

Rx=Qtb. Rx = Q^tb. Rx=Qtb.

Since RRR is an upper triangular matrix, this can be solved directly: denoting xix_ixi​ the iii-th component of xxx and letting b~=Qtb\tilde{b} = Q^tbb~=Qtb, equation (1) can be expanded into

rnnxn=b~nrn−1,n−1xn−1+rn−1,nxn=b~n−1⋮\begin{array}{rcl} r_{nn}x_n &=& \tilde{b}_n\\ r_{n-1,n-1}x_{n-1} + r_{n-1,n}x_n &=& \tilde{b}_{n-1}\\ &\vdots&\end{array}rnn​xn​rn−1,n−1​xn−1​+rn−1,n​xn​​==⋮​b~n​b~n−1​​

which can be iteratively solved with

xk ⁣ ⁣= ⁣ ⁣rkk−1(b~k−∑i=1n−krk,k+ixk+i)fork=n,n−1,...,1. x_{k} \quad\!\! =\quad\!\! r_{kk}^{-1}\left(\tilde{b}_k - \sum_{i=1}^{n-k}r_{k,k+i}x_{k+i}\right)\quad\text{for}\quad k=n,n-1,...,1. xk​=rkk−1​(b~k​−i=1∑n−k​rk,k+i​xk+i​)fork=n,n−1,...,1.

Here's a simple implementation in Julia:

using StableRNGs, LinearAlgebra
rng = StableRNG(111)

n = 5
A = rand(rng, n, n)
b = rand(rng, n)

Q, R = qr(A)
x = zero(b)
b̃ = Q' * b
for k = n:-1:1
    w = b̃[k]
    for i = 1:n-k
        w -= R[k, k+i] * x[k+i]
    end
    x[k] = w / R[k, k]
end

e = round(norm(A * x - b), sigdigits=2)
println("‖Ax-b‖ = $e")
‖Ax-b‖ = 2.8e-16

For pairs (A,b)(A, b)(A,b) with no specific properties, these direct methods typically have complexity O(n3)\mathcal O(n^3)O(n3). In the QR approach above, the factorisation is the most expensive part, solving the triangular system (1) is only O(n2)\mathcal O(n^2)O(n2). LU-based solvers typically have a better complexity constant (about half that of QR) and so are preferred in practice but either way, for nnn "large" (typically, larger than 1000), these methods all become prohibitively expensive.

In what follows, we consider iterative methods which start from a candidate vector x0x_0x0​ and generate a sequence of xkx_kxk​ such that xk→xx_k \to xxk​→x in some sense as kkk increases with the hope that for some kkk sufficiently large (and yet k≪nk\ll nk≪n), the solution is "good enough". Of course each step of such methods should be cheap so that the overall cost of the method to reach a suitable approximation is significantly lower than the O(n3)\mathcal O(n^3)O(n3) of a direct method.

Iterative methods to solve a linear system

We consider iterative methods of the form

xk ⁣ ⁣= ⁣ ⁣xk−1+αkpk ⁣ ⁣= ⁣ ⁣x0+∑i=1kαipi x_{k} \quad\!\! =\quad\!\! x_{k-1} + \alpha_{k} p_{k} \quad\!\! =\quad\!\! x_0 + \sum_{i=1}^{k} \alpha_i p_i xk​=xk−1​+αk​pk​=x0​+i=1∑k​αi​pi​

for some initial candidate x0∈Rnx_0\in\mathbb R^nx0​∈Rn, a set of linearly independent pk∈Rnp_k \in\mathbb R^npk​∈Rn and some coefficients αk∈R\alpha_k\in\mathbb Rαk​∈R, all such that xk→xx_k \to xxk​→x as kkk increases and with xn=xx_n=xxn​=x (since, ultimately, the pkp_kpk​ form a basis of Rn\mathbb R^nRn).

Such a set of vectors pkp_kpk​ can be orthogonalised along the way using the Gram-Schmidt procedure (which we covered in a previous post) so that pk∈span(q1,…,qk)p_k\in \mathrm{span}(q_1,\dots,q_k)pk​∈span(q1​,…,qk​) and (4) can then be expressed as

{qk∝ pk−∑i=1k−1⟨pk,qi⟩qixk=x0+∑i=1kβiqi \begin{cases} q_k &\propto& \,\, p_k - \sum_{i=1}^{k-1} \left\langle p_k, q_i\right\rangle q_i\\ x_{k} &=& x_0 + \sum_{i=1}^k \beta_i q_i \end{cases} {qk​xk​​∝=​pk​−∑i=1k−1​⟨pk​,qi​⟩qi​x0​+∑i=1k​βi​qi​​

for some coefficients βi∈R\beta_i\in\mathbb Rβi​∈R. This amounts to iteratively building an orthonormal basis {q1,…,qk}\{q_1, \dots, q_k\}{q1​,…,qk​} of a subspace of dimension kkk and determining an approximate solution xkx_{k}xk​ in it.

There's two elements that can be adjusted in the above procedure:

  1. how to pick the βi\beta_iβi​,

  2. how to pick the vectors pkp_{k}pk​.

Both of these can be guided by how easy/cheap the iterations are, and how good an approximation xk+1x_{k+1}xk+1​ is to the solution xxx.

Minimum residual criterion

The criterion we'll use on this page is the minimum residual criterion. We will discuss another one in the follow up post. It's a fairly natural criterion that requires taking the βi\beta_iβi​ such that the 2-norm of the residual rk=Axk−br_{k} = Ax_{k} - brk​=Axk​−b is minimised with

∥Axk−b∥2 ⁣ ⁣= ⁣ ⁣∥r0+∑i=1kβiAqi∥2. \|Ax_k - b\|_2 \quad\!\! =\quad\!\! \left\|r_0 + \sum_{i=1}^k\beta_i Aq_i\right\|_2. ∥Axk​−b∥2​=∥∥​r0​+i=1∑k​βi​Aqi​∥∥​2​.

Writing Q(k)Q^{(k)}Q(k) the matrix with columns {q1,…,qk}\{q_1,\dots,q_k\}{q1​,…,qk​} and β(k)\beta^{(k)}β(k) the vector in Rk\mathbb R^kRk with components {β1,…,βk}\{\beta_1,\dots,\beta_k\}{β1​,…,βk​}, the above can be expressed in matrix form as

β(k) ⁣ ⁣= ⁣ ⁣arg⁡min⁡β∈Rk∥r0+AQ(k)β∥2 \beta^{(k)} \quad\!\! =\quad\!\! \arg\min_{\beta\in\mathbb R^k} \quad\left\| r_0 + AQ^{(k)}\beta \right\|_2 β(k)=argβ∈Rkmin​∥∥​r0​+AQ(k)β∥∥​2​

which is just a least-square regression in kkk dimensions. These subsequent regressions can each be solved cheaply as we show below.

Assume we simultaneously orthogonalise the vectors AqiAq_iAqi​ to get another orthonormal basis {q~1,…,q~k}\{\tilde{q}_1,\dots,\tilde{q}_k\}{q~​1​,…,q~​k​}. Let Q~(k)\tilde{Q}^{(k)}Q~​(k) be the matrix with columns q~i\tilde{q}_iq~​i​. Then, at step kkk, we can write AQ(k)=Q~(k)R(k)AQ^{(k)} = \tilde{Q}^{(k)}R^{(k)}AQ(k)=Q~​(k)R(k), and the least-square problem becomes

γ(k) ⁣ ⁣= ⁣ ⁣arg⁡min⁡γ∈Rk∥r0+Q~(k)γ∥2withγ(k)=R(k)β(k). \gamma^{(k)} \quad\!\! =\quad\!\! \arg\min_{\gamma \in \mathbb R^k}\quad \left\| r_0 + \tilde{Q}^{(k)}\gamma \right\|_2 \quad\text{with}\quad \gamma^{(k)} = R^{(k)}\beta^{(k)}. γ(k)=argγ∈Rkmin​∥∥​r0​+Q~​(k)γ∥∥​2​withγ(k)=R(k)β(k).

Since Q~(k)\tilde{Q}^{(k)}Q~​(k) has orthogonal columns, the least-square solution to this problem is simply γ(k)=−(Q~(k)tQ~(k))−1Q~(k)tr0=−Q~(k)tr0\begin{array}{rcl} \gamma^{(k)} &=& -({\tilde{Q}^{(k)}}^t \tilde{Q}^{(k)})^{-1}{\tilde{Q}^{(k)}}^t r_0 \\ &=& -{\tilde{Q}^{(k)}}^t r_0\end{array}γ(k)​==​−(Q~​(k)tQ~​(k))−1Q~​(k)tr0​−Q~​(k)tr0​​ and finding β(k)\beta^{(k)}β(k) such that R(k)β(k)=γ(k)R^{(k)}\beta^{(k)} = \gamma^{(k)}R(k)β(k)=γ(k) can be done directly and cheaply since R(k)R^{(k)}R(k) is upper triangular (using the same iteration as in (3)). Note also that we get the residual rkr_krk​ directly from γ(k)\gamma^{(k)}γ(k) since rk=r0+Q~(k)γ(k)r_k = r_0 + \tilde{Q}^{(k)}\gamma^{(k)}rk​=r0​+Q~​(k)γ(k).

Picking this criterion as well as the "Krylov" directions (see below) leads to the vanilla version of the GMRES algorithm (Generalised Minimal Residual Method).

Generating directions

Ideally, we would like the space Pk\mathcal P_kPk​ spanned by the {p1,…,pk}\{p_1, \dots, p_k\}{p1​,…,pk​} to be such that the projection of xxx on Pk\mathcal P_kPk​ is close to xxx. This, unfortunately, is not very easy to translate into a cheap procedure for generating a good sequence of pkp_kpk​ not least because we don't have access to xxx. After executing step k−1k-1k−1, the only new information we can compute is the new residual rk−1r_{k-1}rk−1​ and so, naturally, we could try using that in forming the new direction pkp_kpk​.

In this point we will briefly discuss three approaches to generating the pkp_kpk​:

  • random, where the pkp_kpk​ are drawn at random e.g. with (pk)i∼N(0,1)(p_k)_i \sim \mathcal N(0, 1)(pk​)i​∼N(0,1),

  • Krylov, where the pkp_kpk​ are the residuals rk−1r_{k-1}rk−1​,

  • gradient, where the pkp_kpk​ are Atrk−1A^tr_{k-1}Atrk−1​.

The first one is not motivated by anything else than "it should work" but should not be expected to work very well, the second one is a common choice, that links with Krylov subspace methods (as discussed in the second part), and finally, the third one is motivated by the gradient of F(x)=∥Ax−b∥22F(x) = \|Ax - b\|_2^2F(x)=∥Ax−b∥22​ which is At(Ax−b)A^t(Ax-b)At(Ax−b) so that

∇F(xk−1) ⁣ ⁣= ⁣ ⁣Atrk−1. \nabla F(x_{k-1}) \quad\!\! =\quad\!\! A^tr_{k-1}. ∇F(xk−1​)=Atrk−1​.

Of course there could be other approaches than these three but they have the advantage of being simple and illustrative.

Note: in a number of important cases, AAA is not built explicitly. Rather we just have a procedure to compute AxAxAx for a given xxx. In such cases it might be impractical to compute AtxA^txAtx explicitly.

Connection to fixed-point methods

In a different context, we had discussed splitting methods where one decomposes the matrix AAA into a sum: A=B+CA=B+CA=B+C for some matrices BBB and CCC to transform the problem into a fixed point equation:

Bx=b−Cx Bx = b - Cx Bx=b−Cx

that can be approached with a damped fixed-point iteration:

Bxk+1=(1−θk)xk+θk(b−Cxk) Bx_{k+1} = (1-\theta_k)x_k + \theta_k ( b - Cx_k) Bxk+1​=(1−θk​)xk​+θk​(b−Cxk​)

for some θk∈(0,1]\theta_k \in (0, 1]θk​∈(0,1]. Picking B=IB= IB=I and C=(A−I)C=(A- I)C=(A−I), the iteration is

xk+1=(1−θk)xk+θk(b+xk−Axk)=xk−θkrk\begin{array}{rcl} x_{k+1} &=& (1-\theta_k)x_k + \theta_k(b + x_k - Ax_k)\\ &=& x_k - \theta_k r_k\end{array}xk+1​​==​(1−θk​)xk​+θk​(b+xk​−Axk​)xk​−θk​rk​​

with rk=Axk−br_k = Ax_k-brk​=Axk​−b. In that sense, the "Krylov" choice earlier can be connected to a fixed-point iteration with a specific decomposition of AAA.

Implementation

We've now discussed the different moving parts and can build a simple implementation which, at step kkk, does the following:

  1. gets a new direction pkp_kpk​ and finds the corresponding orthonormal qkq_kqk​,

  2. computes AqkAq_kAqk​ and finds the corresponding orthonormal q~k\tilde{q}_kq~​k​ as well as the ri,kr_{i,k}ri,k​,

  3. computes γk=−(Q~(k))tr0\gamma_k = - (\tilde{Q}^{(k)})^t r_0γk​=−(Q~​(k))tr0​,

  4. solves R(k)β(k)=γkR^{(k)}\beta^{(k)}=\gamma_kR(k)β(k)=γk​ (a triangular system of size kkk).

For step (1) and (2) we can use the modified Gram-Schmidt procedure.

In the points below, we show code that solves each of these steps and ultimately put them all together to form a working iterative solver. The code is not optimised but should hopefully be easy to read and to analyze.

Modified-GS step

This code computes the kkk-th column of QQQ and RRR in place by orthogonalising a given vector vvv against the first (k−1)(k-1)(k−1) columns of QQQ.

using LinearAlgebra, StableRNGs

function mgs!(Q::AbstractMatrix, R::AbstractMatrix, v::Vector, k::Int)
    n = length(v)
    Q[:, k] .= v
    for i = 1:k-1
        R[i, k] = dot(Q[:, k], Q[:, i])
        Q[:, k] .-= R[i, k] .* Q[:, i]
    end
    Q[:, k] = normalize(Q[:,k])
    R[k, k] = dot(v, Q[:, k])
    return
end

# simple check
begin
    rng = StableRNG(1234)
    n = 5
    A = randn(rng, n, n)
    Q = zeros(n, n)
    R = zeros(n, n)
    for k = 1:n
        mgs!(Q, R, A[:, k], k)
    end
    e1 = round(maximum(abs.(Q'*Q - I)), sigdigits=2)
    e2 = round(maximum(abs.(A - Q*R)), sigdigits=2)
    println("max entry in |Q'Q-I|: $e1")
    println("max entry in |A-QR|: $e2")
end
max entry in |Q'Q-I|: 1.5e-15
max entry in |A-QR|: 3.5e-16

The computational complexity (ignoring constants) of this function is:

  • computation of rikr_{ik}rik​: (k−1)(k-1)(k−1) dot products i.e. O(kn)\mathcal O(kn)O(kn),

  • computation of the entries of qkq_kqk​: O(kn)\mathcal O(kn)O(kn),

  • normalisation of qkq_kqk​ and computation of rkkr_{kk}rkk​: O(n)\mathcal O(n)O(n)

so O(kn)\mathcal O(kn)O(kn) overall.

Upper triangular solve

The code below is just a simple function implementing (3):

function trisolve(R::AbstractMatrix, v::Vector)
    k = size(R, 1)
    β = zero(v)
    for j = k:-1:1
        β[j] = v[j]
        for i = 1:k-j
            β[j] -= R[j, j+i] * β[j+i]
        end
        β[j] /= R[j, j]
    end
    return β
end

# simple check
begin
    rng = StableRNG(414)
    m = 5
    R = randn(rng, m, m)
    for i=1:m, j=1:i-1
        R[i, j] = 0
    end
    β = randn(rng, m)
    v = R * β
    β̂ = trisolve(R, v)
    e = round(norm(β-β̂), sigdigits=2)
    println("‖β-β̂‖: $e")
end

  • for each index jjj compute (k−j)(k-j)(k−j) products and subtractions and one division.

so overall O(k2)\mathcal O(k^2)O(k2) complexity where kkk is the size of vvv.

Core method

We now put all the blocks together to form the itersolve function. In the function you'll see that we keep track of elapsed time, and the norm of the residuals, this is to facilitate analysis and comparison later on.

function itersolve(A::Matrix, b::Vector, x0::Vector;
                   niter::Int=10, dirs=:rand)
    start = time()
    n = size(A, 1)
    r0 = A * x0 - b

    # objects we'll use to store the results
    xk = copy(x0)
    rk = copy(r0)
    pk = zeros(n)
    Q = zeros(n, niter)
    Q̃ = zeros(n, niter)
    R = zeros(niter, niter)
    norm_rk = zeros(niter + 1)
    norm_rk[1] = norm(r0)
    times = zeros(niter+1)
    times[1] = time() - start

    @views for k = 1:niter
        # <O> Obtain a new direction pk
        if dirs == :rand
            pk = randn(n)
        elseif dirs == :krylov
            pk = rk
        elseif dirs == :grad
            pk = A' * rk
        end

        # <A> Orthog. of pk using MGS. We don't care about R[:, k] here,
        # it will be overwritten at the next step anyway
        mgs!(Q, R, pk, k)

        # <B> Orthog. of A*qk using MGS, we do care about R[:, k] here
        mgs!(Q̃, R, A*Q[:, k], k)

        # <C> Solving the least-square problem
        γk = -Q̃[:, 1:k]' * r0

        # <D> Solving the triangular problem
        βk = trisolve(R[1:k, 1:k], γk)

        # <E> Approximation + residuals
        xk .= x0 .+ Q[:, 1:k] * βk
        rk .= r0 .+ Q̃[:, 1:k] * γk

        norm_rk[k+1] = norm(rk)
        times[k+1] = time() - start
    end
    return xk, norm_rk, times
end

# quick check (n steps must lead to the correct solution)
begin
    rng = StableRNG(908)
    n = 5
    A = randn(rng, n, n)
    x = randn(rng, n)
    b = A * x

    _, nrk_random, _ = itersolve(A, b, x0; niter=n, dirs=:rand)

    println("‖Axn - b‖: $norm_rn (random)")

    println("‖Axn - b‖: $norm_rn (krylov)")

    _, nrk_grad, _ = itersolve(A, b, x0; niter=n, dirs=:grad)
    norm_rn = round(nrk_grad[end], sigdigits=2)
    println("‖Axn - b‖: $norm_rn (grad)")
end

The computational complexity (ignoring constants) at iteration kkk of this function is:

  • (O) for grad: one matrix-vector product with AtA^tAt i.e. O(n2)\mathcal O(n^2)O(n2),

  • (A, B) two MGS steps i.e. O(kn)\mathcal O(kn)O(kn)

  • (B) one matrix-vector product with AAA i.e. O(n2)\mathcal O(n^2)O(n2)

  • (C) one matrix-vector product with Q~(k)\tilde{Q}^{(k)}Q~​(k) i.e. O(kn)\mathcal O(kn)O(kn)

  • (D) one upper-triangular solve of a k×kk\times kk×k system i.e. O(k2)\mathcal O(k^2)O(k2)

  • (E) one application of Q(k)Q^{(k)}Q(k) and one of Q~(k)\tilde{Q}^{(k)}Q~​(k) i.e. O(nk)\mathcal O(nk)O(nk)

The step with dominant complexity is the matrix-vector multiplication (computation of AqkAq_kAqk​) (and computation of AtrkA^tr_kAtrk​ in the grad case). Applying a n×nn\times nn×n matrix has complexity O(n2)\mathcal O(n^2)O(n2) making the overall procedure O(Kn2)\mathcal O(Kn^2)O(Kn2) with KKK the number of steps.

Note: in many cases, there is a specific procedure available to compute (exactly or approximately) AxAxAx for some xxx with complexity better than O(n2)\mathcal O(n^2)O(n2). This can for instance be the case when AAA is very sparse, or in some physics problem where the the Fast Multipole Method (FMM) can be used (see e.g. Beatson and Greengard (1997)).

Comparison

In the code below, we consider a random n×nn\times nn×n linear system that is poorly conditioned and check the norm of the residuals ∥rk∥\|r_k\|∥rk​∥ as kkk increases for the different direction choices. We also use the gmres! function from the excellent IterativeSolvers.jl package as a rough sanity check that our code is correct.

Note: this is just meant to be a quick benchmark which highlights some interesting points for discussion, it's not meant to be an in-depth comparison.
Note also that we use random matrices with no or very little structure, so iterative methods need a number of iteration comparable to the dimension to get satisfactory results, for more useful AAA the convergence is typically much faster.
using IterativeSolvers, PyPlot

rng = StableRNG(908)

n = 50
A = randn(rng, n, n)^4 .+ rand(rng, n, n)
x = randn(rng, n)
b = A * x
x0 = randn(rng, n)
r0 = A * x0 - b

κ = round(cond(A), sigdigits=2)
println("cond(A): $κ")

cases = Dict()
for dirs in (:rand, :grad, :krylov)
    cases[dirs] = itersolve(A, b, x0; niter=n, dirs=dirs)
end

figure(figsize=(8, 6))

    plot(cases[dirs][2], label=dirs)

On this first plot we can see a number of interesting things:

  1. the "krylov" directions (where pk=rkp_k = r_kpk​=rk​) leads to identical iterations as those from gmres! which gives some confidence in the implementation of the itersolve function in terms of correctness,

  2. the "random" directions have a generally worse performance as could be expected since there's no information in the direction pkp_kpk​,

  3. the "grad" directions (where pk=Atrkp_k = A^tr_kpk​=Atrk​) lead to much faster convergence than the other two,

  4. all choices eventually lead to ∥rn∥≈0\|r_n\|\approx 0∥rn​∥≈0 as expected.

Note: when writing this post and running experiments, I was a bit surprised by how much better the "grad" version behaves compared to the others here, especially since I had not seen that choice discussed in the literature. It may well be an artefact of the benchmark though (see e.g. next post), or a well known fact. Either way if you have thoughts on this, I'll be glad to hear from you.

We can repeat a similar experiment with a larger matrix and more iterations and look at the time taken since the start instead of the number of iterations. Before doing so note that:

  1. if you try to reproduce these experiments, you should run them a couple of times to make sure the pre-compilation of the function itersolve is factored away,

  2. if you compare the timings with gmres! you'll see that itersolve is significantly slower, this is essentially because its implementation is geared for readability rather than speed,

  3. the main aim of the experiment below is to show that the "grad" choice is still much better here even though each step is twice as expensive as the other two.

rng = StableRNG(9080)

n = 250
# a random matrix with some sparsity
A = 10 * randn(rng, n, n) .* rand(rng, n, n) .* (rand(rng, n, n) .> 0.3)
nz = sum(A .== 0)
sp = round(nz / (n * n) * 100)
println("Sparsity ≈ $sp%")

x = randn(rng, n)
b = A * x
x0 = randn(rng, n)

cases = Dict()
for dirs in (:rand, :grad, :krylov)
    cases[dirs] = itersolve(A, b, x0; niter=n, dirs=dirs)
end

figure(figsize=(8, 6))
for dirs in (:rand, :grad, :krylov)
    semilogy(cases[dirs][3], cases[dirs][2], label=dirs)
end

xlabel("Time elapsed [s]")
ylabel("Norm of the residuals")
legend()
Sparsity ≈ 30.0%

Again, this should not be considered to be a thorough benchmark but one can see that even taking into account the extra time needed per step in the grad case, the convergence is significantly faster in the low iteration count for the "grad" choice.

As a final note, we look at the time taken per method when the matrix size increases and with a fixed number of iteration:

rng = StableRNG(90801)

K = 50
times = Dict(:rand=>[], :grad=>[], :krylov=>[], :gmres=>[])
N = collect(200:100:3_000)
for n in N
    A = randn(rng, n, n)
    x = randn(rng, n)
    b = A*x
    x0 = randn(rng, n)

    for dirs in (:rand, :grad, :krylov)
        start = time()
        itersolve(A, b, x0; niter=K, dirs=dirs)
        push!(times[dirs], time()-start)
    end
    start = time()
    gmres!(copy(x0), A, b, maxiter=K, restart=K+1)
    push!(times[:gmres], time()-start)
end

figure(figsize=(8, 6))
for dirs in (:rand, :grad, :krylov)
    plot(N, times[dirs], label=dirs)
end
plot(N, times[:gmres], label="gmres")
legend()

xlabel("Matrix size")
ylabel("Time taken [s]")

As could be expected, the gmres! function is significantly faster (our code is really not optimised) but all methods exhibit the same behaviour, scaling like n2n^2n2 as expected. Note that only the trend should be looked at, the peaks and trophs should be ignored and are mostly due to how un-optimised our implementation is and how well Julia manages to optimise the steps at various sizes.
