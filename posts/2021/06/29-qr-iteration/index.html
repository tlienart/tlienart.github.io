Diagonalisation

Let A∈Cn×nA \in \mathbb C^{n\times n}A∈Cn×n, we call an eigenpair a nonzero vector v∈Cnv\in\mathbb C^nv∈Cn and a scalar λ∈C\lambda \in \mathbb Cλ∈C if Av=λvAv = \lambda vAv=λv. If we can find nnn linearly independent {v1,…,vn}\{v_1, \dots, v_n\}{v1​,…,vn​} with corresponding {λ1,…,λn}\{\lambda_1, \dots,\lambda_n\}{λ1​,…,λn​} then AAA is said to be diagonalisable and we can write

AV ⁣ ⁣= ⁣ ⁣VΛ AV \quad\!\! =\quad\!\! V\Lambda AV=VΛ

where the columns of VVV are the vkv_kvk​ and Λ\LambdaΛ is a diagonal matrix with Λkk=λk\Lambda_{kk} = \lambda_kΛkk​=λk​. Further, since the vkv_kvk​ are linearly independent, VVV is invertible and

A ⁣ ⁣= ⁣ ⁣VΛV−1. A \quad\!\! =\quad\!\! V\Lambda V^{-1}. A=VΛV−1.

We will focus on determining eigenvalues and eigenvectors of a real-symmetric matrix with distinct eigenvalues which is a simple yet useful base scenario. As we show below, in that case the eigenvalues are real and the eigenvectors are orthogonal which will allow us to connect things nicely with the previous post on Gram-Schmidt orthogonalisation and QR factorisation.

Theorem: A Hermitian matrix AAA (i.e. such that A⋆=AA^\star=AA⋆=A) has real eigenvalues.

Let (v,λ)(v, \lambda)(v,λ) be any eigenpair for AAA so that Av=λvAv = \lambda vAv=λv. Then, since A=A⋆A = A^\starA=A⋆ we have

(Av)⋆v=v⋆A⋆v ⁣ ⁣= ⁣ ⁣v⋆(Av)\begin{array}{rcl} (Av)^\star v &=& v^\star A^\star v \quad\!\! =\quad\!\! v^\star (Av)\end{array}(Av)⋆v​=​v⋆A⋆v=v⋆(Av)​ the left and right term imply λ⋆v⋆v=λv⋆v\begin{array}{rcl} \lambda^\star v^\star v &=& \lambda v^\star v\end{array}λ⋆v⋆v​=​λv⋆v​ and therefore λ=λ⋆\lambda = \lambda^\starλ=λ⋆ since vvv is nonzero.

Theorem: the eigenvectors of a Hermitian matrix AAA corresponding to distinct eigenvalues are orthogonal.

Let (v1,λ1)(v_1, \lambda_1)(v1​,λ1​) and (v2,λ2)(v_2, \lambda_2)(v2​,λ2​) be two eigenpairs with λ1≠λ2\lambda_1\neq \lambda_2λ1​=λ2​; then,

(Av1)⋆(Av2)=λ1λ2v1⋆v2=v1⋆A⋆(λ2v2) ⁣ ⁣= ⁣ ⁣λ22v1⋆v2\begin{array}{rcl} (Av_1)^\star (Av_2) &=& \lambda_1\lambda_2 v_1^\star v_2 \\ &=& v_1^\star A^\star (\lambda_2 v_2) \quad\!\! =\quad\!\! \lambda_2^2 v_1^\star v_2\end{array}(Av1​)⋆(Av2​)​==​λ1​λ2​v1⋆​v2​v1⋆​A⋆(λ2​v2​)=λ22​v1⋆​v2​​

where for the first equality we just used the definition of eigenpairs and that the eigenvalues are real, and the second uses A=A⋆A=A^\starA=A⋆. Now, since λ1≠λ2\lambda_1 \neq \lambda_2λ1​=λ2​ but λ1λ2v1⋆v2=λ22v1⋆v2\lambda_1\lambda_2v_1^\star v_2 = \lambda_2^2 v_1^\star v_2λ1​λ2​v1⋆​v2​=λ22​v1⋆​v2​, we must necessarily have v1⋆v2=0v_1^\star v_2 = 0v1⋆​v2​=0.

Corollary: let AAA be real-valued and symmetric, then all eigenvectors are real-valued.

This is a trivial consequence of the first theorem: since all eigenvalues are real, for any eigenpair we have Av=λvAv = \lambda vAv=λv but since AAA is real-valued and λ\lambdaλ is real, we must necessarily have the entries of vvv be real too.

A last thing we should prove is that a symmetric matrix is always diagonalisable but we'll shove this under the carpet for now and refer the reader to e.g. Golub and Van Loan (1983) for details.

For the rest of this page, we'll assume that AAA is real-valued, symmetric and has distinct (real) eigenvalues λk\lambda_kλk​ with normalised eigenvectors vkv_kvk​ (i.e. the vkv_kvk​ form an orthonormal basis of Rn\mathbb R^nRn). Further, we will assume (without loss of generality) that the (vk,λk)(v_k,\lambda_k)(vk​,λk​) are ordered by the absolute value of λk\lambda_kλk​ so that ∣λ1∣>∣λ2∣>⋯>∣λn∣|\lambda_1|>|\lambda_2|>\dots>|\lambda_n|∣λ1​∣>∣λ2​∣>⋯>∣λn​∣.

The Power Method

Let c∈Rnc \in \mathbb R^nc∈Rn be an arbitrary vector. Since the normalized eigenvectors vkv_kvk​ of AAA form an orthonormal basis, we can write

c ⁣ ⁣= ⁣ ⁣∑i=1nαiviwithαi = ⟨c,vi⟩. c \quad\!\! =\quad\!\! \sum_{i=1}^n \alpha_i v_i \quad\text{with}\quad \alpha_i \,\,=\,\, \left\langle c, v_i\right\rangle. c=i=1∑n​αi​vi​withαi​=⟨c,vi​⟩.

Then, multiplying ccc by the matrix AkA^kAk for k≥1k\ge 1k≥1, we have:

Akc ⁣ ⁣= ⁣ ⁣∑i=1nαiλikvi ⁣ ⁣= ⁣ ⁣λ1k[α1v1+∑i=2nαi(λiλ1)kvi]. A^kc \quad\!\! =\quad\!\! \sum_{i=1}^n \alpha_i \lambda_i^k v_i \quad\!\! =\quad\!\! \lambda_1^k\left[\alpha_1 v_1 + \sum_{i=2}^n \alpha_i \left({\lambda_i\over \lambda_1}\right)^k v_i\right]. Akc=i=1∑n​αi​λik​vi​=λ1k​[α1​v1​+i=2∑n​αi​(λ1​λi​​)kvi​].

All the ratios ∣λi/λ1∣|\lambda_i/\lambda_1|∣λi​/λ1​∣ are smaller than 1 and so vanish for large kkk since we assumed that ∣λ1∣>∣λi∣|\lambda_1|>|\lambda_i|∣λ1​∣>∣λi​∣ for i=2,…,ni=2,\dots,ni=2,…,n. Let us assume that we picked ccc such that α1≠0\alpha_1 \neq 0α1​=0; then, for kkk sufficiently large, AkcA^k cAkc becomes dominated by α1λ1kv1\alpha_1\lambda_1^k v_1α1​λ1k​v1​ as all other terms go to zero.

In summary, for a ccc such that ⟨c,v1⟩≠0\left\langle c, v_1\right\rangle \neq 0⟨c,v1​⟩=0, AkcA^kcAkc will align with v1v_1v1​ as kkk gets larger. Using v∝wv\propto wv∝w to mean v=w/∥w∥v = w / \|w\|v=w/∥w∥ for nonzero www, we can use the following iteration:

  • w(1)∝Acw^{(1)} \propto Acw(1)∝Ac

  • w(k)∝Aw(k−1)w^{(k)} \propto Aw^{(k-1)}w(k)∝Aw(k−1) for k=2,…k=2,\dotsk=2,…

for sufficiently large kkk, w(k)w^{(k)}w(k) is approximately aligned with v1v_1v1​. Correspondingly, we will have Aw(k)≈λ1w(k)Aw^{(k)} \approx \lambda_1 w^{(k)}Aw(k)≈λ1​w(k) and can recover an approximation of λ1\lambda_1λ1​ considering the Raleigh quotient:

ρ(k) ⁣ ⁣= ⁣ ⁣⟨Aw(k),w(k)⟩⟨w(k),w(k)⟩ \rho^{(k)} \quad\!\! =\quad\!\! { \left\langle Aw^{(k)}, w^{(k)}\right\rangle \over \left\langle w^{(k)}, w^{(k)}\right\rangle } ρ(k)=⟨w(k),w(k)⟩⟨Aw(k),w(k)⟩​

and ρ(k)→λ1\rho^{(k)} \to \lambda_1ρ(k)→λ1​ with increasing kkk.

Implementation

The power method is very simple to implement, here's a basic implementation in Julia which shows convergence:

using LinearAlgebra, StableRNGs

function power_method(A::Symmetric, x0; iter=10)
    λ_max = eigmax(A)   # to show convergence
    w = copy(x0)
    Aw = zero(w)
    ρ = zero(eltype(w))
    for i = 1:iter
        Aw .= A * w
        ρ  = dot(Aw, w) / dot(w, w)
        w .= normalize(Aw)
        println("Step $i: ", round(abs((ρ - λ_max) ./ λ_max), sigdigits=2))
    end
    return w, ρ
end

rng = StableRNG(555)
n = 5
A = Symmetric(rand(rng, n, n))
c = randn(rng, n)

w, ρ = power_method(A, c)
err_eig = round(norm(A * w - ρ * w), sigdigits=2)
println("‖Aw - ρw‖: $err_eig")

From the Power Method to the QR iteration

Power-Method + projection

The power method as shown above can fairly easily give us (λ1,v1)(\lambda_1, v_1)(λ1​,v1​) but what about the other eigenpairs? Let c1c_1c1​ be an arbitrary vector with ⟨c1,v1⟩≠0\left\langle c_1, v_1\right\rangle \neq 0⟨c1​,v1​⟩=0, we've shown that the power method leads to w1(k)≈v1w_1^{(k)} \approx v_1w1(k)​≈v1​ for suitably large kkk. We could subtract the projection of c1c_1c1​ onto the space spanned by w1(k)w_1^{(k)}w1(k)​ and apply the power method to try to get v2v_2v2​. Indeed, with c1=∑i=1n⟨c1,vi⟩vic_1 = \sum_{i=1}^n \left\langle c_1, v_i\right\rangle v_ic1​=∑i=1n​⟨c1​,vi​⟩vi​, we'd have

c2=c1−⟨c1,w1(k)⟩w1(k)≈∑i=2n⟨c1,vi⟩vi\begin{array}{rcl} c_2 &=& c_1 - \left\langle c_1, w_1^{(k)}\right\rangle w_1^{(k)}\\ &\approx& \sum_{i=2}^n \left\langle c_1, v_i\right\rangle v_i\end{array}c2​​=≈​c1​−⟨c1​,w1(k)​⟩w1(k)​∑i=2n​⟨c1​,vi​⟩vi​​

and with the same development as that of the previous point, we get that Akc2∝v2A^kc_2 \propto v_2Akc2​∝v2​ (approximately). We could iterate this approach until we've approximated all vnv_nvn​:

  • take cj=cj−1−⟨cj−1,wj−1(k)⟩wj−1(k)c_j = c_{j-1} - \left\langle c_{j-1}, w^{(k)}_{j-1}\right\rangle w^{(k)}_{j-1}cj​=cj−1​−⟨cj−1​,wj−1(k)​⟩wj−1(k)​

  • let wj(k)∝Akcjw^{(k)}_j \propto A^kc_jwj(k)​∝Akcj​

for sufficiently large kkk, each of the wj(k)w^{(k)}_jwj(k)​ would be approximately equal to vjv_jvj​. Note that we wouldn't have to use the same kkk at every power-step and could just use whatever kkk is large enough to reach some notion of convergence.

Enters the QR factorisation

We could try improving on the previous procedure so that we compute all the vectors simultaneously. For this, consider an arbitrary matrix C1∈Rn×nC_1 \in \mathbb R^{n\times n}C1​∈Rn×n instead of a single vector ccc and repeatedly apply the following steps:

  1. compute Mk=ACkM_k = AC_kMk​=ACk​

  2. extract an orthonormal basis QkQ_kQk​ out of MkM_kMk​ via the QR algorithm i.e, Mk=QkRkM_k = Q_kR_kMk​=Qk​Rk​

  3. let Ck+1=QkC_{k+1}=Q_kCk+1​=Qk​ and go to step 1.

this amounts effectively to the same steps as before but instead of going for all vector index jjj and then for each power index kkk, we go for each power index kkk and for all vector index jjj.

A specific C1C_1C1​ we could use of course is Q1=QQ_1=QQ1​=Q, the factor of A=QRA=QRA=QR with then:

  1. compute Mk=AQk−1M_k = AQ_{k-1}Mk​=AQk−1​ for k=2,…k=2, \dotsk=2,…,

  2. compute the QR factorisation of Mk=QkRkM_k = Q_k R_kMk​=Qk​Rk​.

Taking a look at the first few iterations, we have:

A=Q1R1M2=AQ1 ⁣ ⁣= ⁣ ⁣Q2R2M3=AQ2 ⁣ ⁣= ⁣ ⁣Q3R3\begin{array}{rcl} A &=& Q_1 R_1 \\ M_2 &=& AQ_1 \quad\!\! =\quad\!\! Q_2 R_2 \\ M_3 &=& AQ_2 \quad\!\! =\quad\!\! Q_3 R_3\end{array}AM2​M3​​===​Q1​R1​AQ1​=Q2​R2​AQ2​=Q3​R3​​ and so on. The second line can also be written Q1R1Q1=Q2R2Q_1R_1Q_1 = Q_2R_2Q1​R1​Q1​=Q2​R2​ or

R1Q1 ⁣ ⁣= ⁣ ⁣Q12R2 R_1Q_1 \quad\!\! =\quad\!\! Q_{12}R_2 R1​Q1​=Q12​R2​

with Q12=Q1tQ2Q_{12} = Q_1^t Q_2Q12​=Q1t​Q2​ (which is still orthonormal). In a similar fashion, we can massage the third line into:

A(Q1Q1t)Q2 ⁣ ⁣= ⁣ ⁣Q3R3 A(Q_1 Q_1^t)Q_2 \quad\!\! =\quad\!\! Q_3 R_3 A(Q1​Q1t​)Q2​=Q3​R3​

or Q2R2Q12=Q3R3Q_2R_2Q_{12} = Q_3R_3Q2​R2​Q12​=Q3​R3​ which we can also write as

R2Q12=Q23R3 R_2Q_{12} = Q_{23}R_3 R2​Q12​=Q23​R3​

with Q23=Q2tQ3Q_{23} = Q_2^tQ_3Q23​=Q2t​Q3​. Bootstrapping from there, we can write RkQk−1,k=Qk,k+1Rk+1R_kQ_{k-1,k} = Q_{k,k+1}R_{k+1}Rk​Qk−1,k​=Qk,k+1​Rk+1​ with Q01=Q1Q_{01}=Q_1Q01​=Q1​ and it's easy to show that Q1…QkQk,k+1=Qk+1Q_1\dots Q_k Q_{k,{k+1}} = Q_{k+1}Q1​…Qk​Qk,k+1​=Qk+1​.

The advantage of expressing the whole iteration in terms of orthonormal matrices is that it's more numerically stable than repeatedly applying AAA which can be poorly conditioned. Writing it out, we have:

  1. get (Qk,k+1,Rk+1)=qr(RkQk−1,k)(Q_{k,k+1}, R_{k+1}) = \text{qr}(R_{k} Q_{k-1,k})(Qk,k+1​,Rk+1​)=qr(Rk​Qk−1,k​) with Q01,R1=qr(A)Q_{01}, R_1=\text{qr}(A)Q01​,R1​=qr(A),

  2. compute Qk+1=QkQk,k+1Q_{k+1} = Q_k Q_{k,k+1}Qk+1​=Qk​Qk,k+1​.

This is the (basic) QR iteration algorithm, also known as the Francis QR algorithm (Golub and Van Loan (1983)). For kkk sufficiently large, the columns of Qk+1Q_{k+1}Qk+1​ will align with the eigenvectors of AAA. Correspondingly, we get the eigenvalues with

Qk+1tAQk+1 ⁣ ⁣≈ ⁣ ⁣diag(λ1,…,λn). Q_{k+1}^tAQ_{k+1} \quad\!\! \approx\quad\!\! \text{diag}(\lambda_1, \dots, \lambda_n). Qk+1t​AQk+1​≈diag(λ1​,…,λn​).

Implementation

A basic version of the QR algorithm is fairly easy to implement as shown below with an implementation in Julia.

function francis_qr(A::Symmetric; iter=20)
    λ = sort(eigvals(A), by=abs, rev=true)   # to show convergence
    Q̃, R̃ = qr(A)
    Q = copy(Q̃)
    D = zero(Q)
    for i = 1:iter
        Q̃, R̃ = qr(R̃ * Q̃)    # step 1
        Q *= Q̃              # step 2
        # computations to show convergence
        D  = Q' * A * Q
        λ̂  = diag(D)
        δ  = round(maximum(abs.((λ̂ .- λ) ./ λ)), sigdigits=2)
        println("Step $i: $δ")
    end
    return Q, D
end

rng = StableRNG(510)
n = 5
A = Symmetric(rand(rng, n, n))

Q, D = francis_qr(A)
Λ = Diagonal(D)

err_offdiag = round(maximum(abs.(D - Λ)), sigdigits=2)
println("|D-Λ|: $err_offdiag")

err_diag = round(maximum(abs.(A * Q - Q * Λ)), sigdigits=2)
println("|AQ-QΛ|: $err_diag")

Generalisations

Here we only considered a fairly simple case (symmetric matrix with distinct eigenvalues). In practice, the QR algorithm is more sophisticated, can deal with non-symmetric matrices and encourage convergence by introducing shifts in the iteration.

See Golub and Van Loan (1983) for a much more detailed approach on the topic, Townsend (2019) is also a nice tutorial discussing the QR algorithm and its shifted variants.

Short references

  1. Golub, Van Loan, Matrix Computations, 1983. – Chapter 7 covers the QR algorithm and chapter 8 considers optimisations for the symmetric case.

  2. Persson, The QR algorithm II, 2006. – A slide deck on the QR algorithm shifted QR and numerical stability.

  3. Townsend, The QR algorithm, 2019. – A tutorial on the QR algorithm using Julia and discussing the shifted QR algorithm.
