Thoughts on first order descent methods

optimisation, projected gradient descent, prospective

This is an experimental set of notes, take it with appropriate care.

First order methods (FOM) broadly designate iterative methods for continuous and (sub)differentiable optimisation that mainly use information from the (sub)gradient of the function.

In these notes we consider again the constrained minimisation problem min⁡x∈Cf(x)\min_{x\in C} f(x)minx∈C​f(x) and, to simplify the presentation, we'll assume that CCC is closed and convex and that fff is strictly convex and smooth on CCC.

What we're interested in, at a high level, is how to generate a minimising sequence for fff i.e., a sequence {xk}\{x_k\}{xk​} with xk∈Cx_k\in Cxk​∈C such that f(xk+1)<f(xk)f(x_{k+1}) < f(x_k)f(xk+1​)<f(xk​) and f(xk)→f(x†)f(x_k) \to f(x^\dagger)f(xk​)→f(x†) as kkk grows.

Remark: all norms ∥⋅∥\|\cdot\|∥⋅∥ on this page are 2-norms.

Local linearisation

Local linearisation is basically what first order methods are about: form a linear approximation of the function around a current point and use that to move towards a promising direction. Still, let's try to look at this from scratch.

Consider the local linearisation of fff around a point a∈C∘a\in C^\circa∈C∘:

f(x) ⁣ ⁣= ⁣ ⁣f(a)+⟨x−a,∇f(a)⟩+r(x,a) f(x) \quad\!\! =\quad\!\! f(a) + \left\langle x-a, \nabla f(a)\right\rangle + r(x, a) f(x)=f(a)+⟨x−a,∇f(a)⟩+r(x,a)

where r(x,a)r(x, a)r(x,a) is the remainder function.

Note: here you may be thinking: Taylor expansion. But let's actually put that on the side for now and just assume that you're building this f(a)+⟨x−a,∇f(a)⟩f(a)+\left\langle x-a,\nabla f(a)\right\ranglef(a)+⟨x−a,∇f(a)⟩ and that you want to investigate the properties of the remainder function.

That function enjoys the following properties:

  1. r(a,a)=0r(a, a)=0r(a,a)=0, and r(x,a)>0r(x, a)>0r(x,a)>0 for all x≠ax\neq ax=a by strict convexity of fff,

  2. r(⋅,a)r(\cdot, a)r(⋅,a) is also strictly convex and smooth for all a∈C∘a\in C^\circa∈C∘.

Let us introduce a more compact notation: ra(δ):=r(a+δ,a)r_a(\delta) := r(a+\delta, a)ra​(δ):=r(a+δ,a) which will be quite useful. It is easy to note that rar_ara​ is smooth and strictly convex with ra(0)=0r_a(0)=0ra​(0)=0 and, in fact, is globally minimised at 000. By definition of strict convexity, we have

ra(δ′) ⁣ ⁣> ⁣ ⁣ra(δ)+⟨δ′−δ,∇ra(δ)⟩,∀δ′≠δ. r_a(\delta') \quad\!\! >\quad\!\! r_a(\delta) + \left\langle \delta'-\delta, \nabla r_a(\delta)\right\rangle, \quad \forall\delta' \neq \delta. ra​(δ′)>ra​(δ)+⟨δ′−δ,∇ra​(δ)⟩,∀δ′=δ.

Taking δ′=0\delta'=0δ′=0 and rearranging terms yields

ra(δ) ⁣ ⁣< ⁣ ⁣⟨δ,∇ra(δ)⟩ ⁣ ⁣≤ ⁣ ⁣∥δ∥∥∇ra(δ)∥, r_a(\delta) \quad\!\! <\quad\!\! \left\langle \delta, \nabla r_a(\delta)\right\rangle \quad\!\! \le\quad\!\! \|\delta\|\|\nabla r_a(\delta)\|, ra​(δ)<⟨δ,∇ra​(δ)⟩≤∥δ∥∥∇ra​(δ)∥,

using Cauchy-Schwartz for the second inequality. Rearranging again gives ra(δ)/∥δ∥<∥∇ra(δ)∥r_a(\delta)/\|\delta\| < \|\nabla r_a(\delta)\|ra​(δ)/∥δ∥<∥∇ra​(δ)∥ for all δ≠0\delta\neq 0δ=0. Since rar_ara​ is smooth away from 000 and since ∇ra(0)=0\nabla r_a(0)=0∇ra​(0)=0 (minimiser), we can take the limit as ∥δ∥→0\|\delta\|\to 0∥δ∥→0 and get the following result.

The function rar_ara​ is o(∥δ∥)o(\|\delta\|)o(∥δ∥) meaning

This could have been obtained directly from the Taylor expansion of fff but I think it's nice to obtain it using only notions of convexity. Another note is that the reasoning essentially still holds if fff is only convex and sub-differentiable.

Admissible descent steps

Let's consider a point xxx in CCC and a step from xxx to x+δx+\deltax+δ for some δ≠0\delta\neq 0δ=0. We're interested in determining what are "good" steps δ\deltaδ to take. Using the notations from the previous point, we have

f(x+δ) ⁣ ⁣= ⁣ ⁣f(x)+⟨δ,∇f(x)⟩+rx(δ). f(x+\delta) \quad\!\! =\quad\!\! f(x)+\left\langle \delta, \nabla f(x)\right\rangle + r_x(\delta). f(x+δ)=f(x)+⟨δ,∇f(x)⟩+rx​(δ).

Local update schemes

What we would like is thus to consider a problem that is simpler than (8) and yet still generates an admissible descent direction (and iterate). A natural way to try to do just that is to replace rx(δ)r_x(\delta)rx​(δ) by a proxy function dx(δ)d_x(\delta)dx​(δ) enjoying the same properties of positive definiteness and strict convexity. The corresponding approximate problem is then

δ~β ⁣ ⁣∈ ⁣ ⁣arg⁡min⁡δ∣x+δ∈C[⟨δ,∇f(x)⟩+βdx(δ)]. \tilde\delta_\beta \quad\!\! \in\quad\!\! \arg\min_{\delta \mid x+\delta \in C} \left[\left\langle \delta, \nabla f(x)\right\rangle + \beta d_x(\delta)\right]. δ~β​∈argδ∣x+δ∈Cmin​[⟨δ,∇f(x)⟩+βdx​(δ)].

Let's now show that these problems can lead to admissible descent steps for the original problem. We can follow a similar reasoning to that which led us to show the non-degeneracy of Dx\mathcal D_xDx​. In particular, observe that as β→∞\beta\to\inftyβ→∞, ∥δ~β∥→0\|\tilde\delta_{\beta}\|\to 0∥δ~β​∥→0. Hence, there exists a β∙\beta^\bulletβ∙ large enough such that for any ν≥β∙\nu \ge \beta^\bulletν≥β∙, ∥δ~ν∥\|\tilde\delta_\nu\|∥δ~ν​∥ is small enough for δ~ν\tilde\delta_\nuδ~ν​ to be in Dx\mathcal D_xDx​.

GPGD is back

Now that we know that (9) can lead to an admissible step, we can suggest iterating over the problem with a sequence of {βk}\{\beta_k\}{βk​}:

δ~βk ⁣ ⁣∈ ⁣ ⁣arg⁡min⁡δ∣xk+δ∈C [⟨δ,∇f(xk)⟩+βkdx(δk)]. \tilde\delta_{\beta_k} \quad\!\! \in\quad\!\! \arg\min_{\delta\mid x_k+\delta \in C} \,\, \left[\left\langle \delta, \nabla f(x_k)\right\rangle + \beta_kd_x(\delta_k)\right].δ~βk​​∈argδ∣xk​+δ∈Cmin​[⟨δ,∇f(xk​)⟩+βk​dx​(δk​)].

However, basic manipulation of that expression show that this is in fact the generalised projected gradient descent (GPGD) that we saw before.

The generalised projected gradient descent (GPGD) corresponds to the following iteration:

xk+1 ⁣ ⁣∈ ⁣ ⁣arg⁡min⁡x∈C{⟨x,∇f(xk)⟩+1αkd(x,xk)} x_{k+1} \quad\!\! \in\quad\!\! \arg\min_{x\in C} \left\{\left\langle x, \nabla f(x_k)\right\rangle + {1\over \alpha_k}d(x, x_k)\right\} xk+1​∈argx∈Cmin​{⟨x,∇f(xk​)⟩+αk​1​d(x,xk​)}

for some αk>0\alpha_k>0αk​>0 and for any positive-definite function ddd that is strictly convex in its first argument. It generates a minimising sequence for fff provided the αk\alpha_kαk​ are small enough.

This may seem like it's not telling us that much, in particular it should be clear that you could pick the αk\alpha_kαk​ infinitesimally small, that it would indeed give you a minimising sequence but also that it wouldn't bring you very far. So at this point there's two comments we can make:

  1. ideal αk\alpha_kαk​ encapsulate a tradeoff between leading to steps that are too big and may not be admissible an steps that are too small to provide useful improvement,

  2. a key element that should hopefully be obvious by now corresponds to how we can interpret ddd: if we know nothing about the function at hand, we can just use a default ∥⋅∥22\|\cdot\|_2^2∥⋅∥22​ but if we do know something useful about the function (and, in fact, about CCC), then that could be encoded in ddd.

Choice of distance and problem structure

The second point is very important: it should be clear to you that you'd want the local problems to be as informed as possible while at the same time you'd want the iterations to not be overly expensive to compute, two extreme cases being:

  • d(x,xk)/αk=∥x−xk∥22/2αkd(x, x_k)/\alpha_k = \|x-x_k\|_2^2/{2\alpha_k}d(x,xk​)/αk​=∥x−xk​∥22​/2αk​ and αk\alpha_kαk​ small, the iterations are cheap to compute but potentially quite poor at decreasing the function, many steps are needed, minimal use of problem structure,

  • d(x,xk)/αk=r(x,xk)d(x, x_k)/\alpha_k = r(x, x_k)d(x,xk​)/αk​=r(x,xk​), the iteration is maximally expensive but only a single step is needed; maximal use of problem structure.

This key tradeoff can be exposed in most iterative methods using gradient information that you'll find in the literature.
