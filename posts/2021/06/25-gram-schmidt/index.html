Orthonormal basis

A set {u1,…,un}\{u_1, \dots, u_n\}{u1​,…,un​} of vectors in Rn\mathbb R^nRn is an orthonormal basis of Rn\mathbb R^nRn if ⟨ui,uj⟩=δij\left\langle u_i, u_j\right\rangle=\delta_{ij}⟨ui​,uj​⟩=δij​ for any i,j∈{1,…,n}i,j \in \{1,\dots, n\}i,j∈{1,…,n}. For such a set, any vector www in Rn\mathbb R^nRn can be expressed as a linear combination of the uiu_iui​:

w ⁣ ⁣= ⁣ ⁣∑i=1nαiui, w \quad\!\! =\quad\!\! \sum_{i=1}^n \alpha_i u_i, w=i=1∑n​αi​ui​,

where the weights αi\alpha_iαi​ can be easily obtained by leveraging the orthonormality: ⟨w,uj⟩=⟨∑i=1nαiui,uj⟩=∑i=1nαiδij ⁣ ⁣= ⁣ ⁣αj.\begin{array}{rcl} \left\langle w, u_j\right\rangle &=& \left\langle \sum_{i=1}^n\alpha_i u_i, u_j\right\rangle \\ &=& \sum_{i=1}^n\alpha_i \delta_{ij} \quad\!\! =\quad\!\! \alpha_j.\end{array}⟨w,uj​⟩​==​⟨∑i=1n​αi​ui​,uj​⟩∑i=1n​αi​δij​=αj​.​

Denoting UUU the n×nn \times nn×n matrix with columns {u1,…,un}\{u_1, \dots, u_n\}{u1​,…,un​}, the above can be written more compactly as UUt=IUU^t=IUUt=I, and

w ⁣ ⁣= ⁣ ⁣Ua⟺a ⁣ ⁣= ⁣ ⁣Utw, w \quad\!\! =\quad\!\! Ua \quad\Longleftrightarrow\quad a \quad\!\! =\quad\!\! U^tw, w=Ua⟺a=Utw,

with a∈Rna \in \mathbb R^na∈Rn the vector whose jjj-th component is αj\alpha_jαj​.

Constructing an orthonormal basis out of a set of independent vectors

Let's say we have {v1,…,vn}\{v_1, \dots, v_n\}{v1​,…,vn​}, a set of nnn  linearly independent vectors in Rn\mathbb R^nRn. We can construct an orthonormal basis {q1,…,qn}\{q_1, \dots, q_n\}{q1​,…,qn​} iteratively by starting with

q1 ⁣ ⁣∝ ⁣ ⁣v1 q_1 \quad\!\! \propto\quad\!\! v_1 q1​∝v1​

where ∝\propto∝ is used to mean that the left-hand-side is proportional to the right-hand-side and has euclidean norm 1 (i.e. here q1=v1/∥v1∥q_1 = v_1/\|v_1\|q1​=v1​/∥v1​∥, note that v1v_1v1​ cannot be zero otherwise the set wouldn't be independent).

The next vector, q2q_2q2​, should be orthogonal to q1q_1q1​ and should span Im{v1,v2}\mathrm{Im}\{v_1, v_2\}Im{v1​,v2​}. Assume we have that q2q_2q2​ for now. We can then write:

v2=α21q1+α22q2\begin{array}{rcl} v_2 &=& \alpha_{21} q_1 + \alpha_{22} q_2\end{array}v2​​=​α21​q1​+α22​q2​​

From the previous point, we know that α21=⟨v2,q1⟩\alpha_{21} = \left\langle v_2, q_1\right\rangleα21​=⟨v2​,q1​⟩ so that

q2 ⁣ ⁣∝ ⁣ ⁣v2−⟨v2,q1⟩q1. q_2 \quad\!\! \propto\quad\!\! v_2 - \left\langle v_2, q_1\right\rangle q_1. q2​∝v2​−⟨v2​,q1​⟩q1​.

The term ⟨v2,q1⟩q1\left\langle v_2, q_1\right\rangle q_1⟨v2​,q1​⟩q1​ is the projection of v2v_2v2​ onto the space spanned by q1q_1q1​.

Iterating this is the Gram-Schmidt procedure (GS):

qk ⁣ ⁣∝ ⁣ ⁣vk−∑i=1k−1⟨vk,qi⟩qi\begin{array}{rcl} q_k \quad\!\! \propto\quad\!\! v_k - \displaystyle\sum_{i=1}^{k-1}\left\langle v_k, q_i\right\rangle q_i \end{array}qk​∝vk​−i=1∑k−1​⟨vk​,qi​⟩qi​​

In words, to construct the kkk-th direction of the basis, we take vkv_kvk​ and subtract is projection onto the space spanned by {q1,…,qk−1}\{q_1, \dots, q_{k-1}\}{q1​,…,qk−1​}.

Side note on the QR and Cholesky decomposition

After the kkk-th step of GS, we can express the vector vkv_kvk​ as a linear combination of the {q1,…,qk}\{q_1,\dots,q_k\}{q1​,…,qk​}:

vk ⁣ ⁣= ⁣ ⁣∑i=1k⟨vk,qi⟩qi. v_k \quad\!\! =\quad\!\! \sum_{i=1}^k \left\langle v_k, q_i\right\rangle q_i. vk​=i=1∑k​⟨vk​,qi​⟩qi​.

Let QQQ be the matrix with columns {q1,…,qk}\{q_1,\dots,q_k\}{q1​,…,qk​} then the above expression can be written vk=Qrkv_k = Qr_kvk​=Qrk​ where rkr_krk​ is a vector of nnn components with iii-th component given by

(rk)i ⁣ ⁣= ⁣ ⁣{ ⟨vk,qi⟩fori∈{1,…,k} 0fori∈{k+1,…,n}. (r_{k})_i \quad\!\! =\quad\!\! \begin{cases} \,\left\langle v_k, q_i\right\rangle &\text{for}\quad i \in \{1, \dots, k\}\\ \quad\,\, 0 &\text{for}\quad i \in \{k+1, \dots, n\}\end{cases}. (rk​)i​={⟨vk​,qi​⟩0​fori∈{1,…,k}fori∈{k+1,…,n}​.

In matrix form, we can write

V ⁣ ⁣= ⁣ ⁣QR V \quad\!\! =\quad\!\! QR V=QR

where VVV is the n×nn\times nn×n matrix with columns {v1,…,vn}\{v_1, \dots, v_n\}{v1​,…,vn​} and RRR is an n×nn\times nn×n upper triangular matrix with rik=⟨vk,qi⟩1i≤kr_{ik} = \left\langle v_k, q_i\right\rangle 1_{i\le k}rik​=⟨vk​,qi​⟩1i≤k​.

Such a decomposition of a full-rank matrix into a product of an orthogonal matrix QQQ and an upper-triangular matrix RRR is called a QR-decomposition or QR-factorisation.

Note: while the GS procedure is one way of obtaining a QR-factorisation of a matrix, it is not the only one and usually not the one that is used in practice as other methods (Householder, Givens) have better computational properties see e.g. Golub and Van Loan (1983).

A final observation is that, starting from (10), we have:

VtV ⁣ ⁣= ⁣ ⁣RtR. V^t V \quad\!\! =\quad\!\! R^tR. VtV=RtR.

Provided the diagonal elements of RRR are positive, the RRR matrix is thus also the Cholesky factor of VtVV^tVVtV. We can always constrain the diagonal elements of RRR to be positive since rii=⟨vi,qi⟩r_{ii} = \left\langle v_i, q_i\right\ranglerii​=⟨vi​,qi​⟩, and we can just swap the sign of qiq_iqi​ after computing (7).

What if the original vectors are not independent?

Let's now say we have {v1,…,vm}\{v_1, \dots, v_m\}{v1​,…,vm​} a set of mmm vectors in Rn\mathbb R^nRn that are not necessarily linearly independent. Let p≤min⁡(m,n)p \le \min(m, n)p≤min(m,n) the dimension of the space spanned by these vectors, we can apply GS to find {q1,…,qp}\{q_1, \dots, q_p\}{q1​,…,qp​} an orthonormal basis of that space. The procedure is identical except some iterations where the right-hand-side of (7) is zero (i.e. vkv_kvk​ can be fully represented by the ongoing set of qjq_jqj​), in which case we just skip that step.

More explicitly, the steps are:

  1. q1∝v1q_1 \propto v_1q1​∝v1​; let c=2,k=2c=2, k=2c=2,k=2

  2. compute w=vk−∑i=1c−1⟨vk,qi⟩qiw = v_k - \sum_{i=1}^{c-1}\left\langle v_k, q_i\right\rangle q_iw=vk​−∑i=1c−1​⟨vk​,qi​⟩qi​

    1. if w=0w=0w=0, increment kkk (the vector vkv_kvk​ can already be represented by the running qiq_iqi​)

    2. otherwise, let qc∝wq_c \propto wqc​∝w and increment both ccc and kkk

Once the last vkv_kvk​ has been seen, we have a set of ppp orthogonal vectors {q1,…,qp}\{q_1, \dots, q_p\}{q1​,…,qp​} that forms a basis in which we can represent all of the vkv_kvk​.

Computational complexity of GS

Say we have mmm vectors in Rn\mathbb R^nRn and want to form an orthonormal basis for those. At step ccc of the algorithm, we have to:

  • compute (c−1)(c-1)(c−1) dot products i.e. O(cn)\mathcal O(cn)O(cn) flops,

  • compute ccc sums of vectors i.e. O(cn)\mathcal O(cn)O(cn) flops,

  • normalise a vector i.e. O(n)\mathcal O(n)O(n) flops

i.e. O(cn)\mathcal O(cn)O(cn) per step. We have to consider all mmm vectors (unless m>nm > nm>n and ccc reaches nnn before kkk in which case we can stop early). To simplify let's say we consider all of them, so that we sum over c=1,…,mc=1,\dots,mc=1,…,m. Then the overall complexity is O(nm2)\mathcal O(nm^2)O(nm2).

Implementing GS

We can code a simple version of GS in Julia, transcribing the core loop explicitly and not caring about allocations or optimisations so that the code is very simple to read:

using LinearAlgebra

function gs(A::Matrix{T}; tol=1e-10) where T
    n, m = size(A)
    Q = zeros(n, n)
    R = zeros(n, m)
    c = 1
    for k = 1:m
        w = A[:, k]
        for i = 1:c-1
            R[i, k] = dot(A[:, k], Q[:, i])
            w .-= R[i, k] .* Q[:, i]            # w = a_k - ∑⟨a_k, q_i⟩q_i
        end
        η = norm(w)
        η < tol && continue                     # check if w ≈ 0
        Q[:, c] .= w ./ η
        R[c, k] = dot(A[:, k], Q[:, c])
        c += 1
    end
    p = c - 1
    return Q[:, 1:p], R[1:p, :]
end

Note: if you're not used to Julia, you might wonder about the dots (.) preceding operators in the code above. They generally mean "element-wise operation" so for instance w .-= v effectively means w[i] -= v[i] for each i. See also this excellent post on the topic.

We can quickly check this works as expected:

Numerical stability and modified-GS

The procedure above (often referred to as classical Gram-Schmidt or CGS) is not numerically stable in that floating-point errors in computation of the qiq_iqi​ will compound badly in the expression (7). We won't do the stability analysis in details, see for instance Björck (2010).

An alternative algorithm is the modified Gram-Schmidt or MGS where (7) is re-organised as an iteration:

  • w(1)=vk−⟨vk,q1⟩q1w^{(1)} = v_k - \left\langle v_k, q_1\right\rangle q_1w(1)=vk​−⟨vk​,q1​⟩q1​,

  • w(i)=w(i−1)−⟨w(i−1),qi−1⟩qi−1w^{(i)} = w^{(i-1)} - \left\langle w^{(i-1)}, q_{i-1}\right\rangle q_{i-1}w(i)=w(i−1)−⟨w(i−1),qi−1​⟩qi−1​ for i=2,…,k−1i=2, \dots, k-1i=2,…,k−1.

In exact arithmetic, this is exactly the same as (7), but in the presence of numerical errors, each w(1)w^{(1)}w(1) is actually obtained as w(1)+e1w^{(1)}+e_1w(1)+e1​ for some small error vector e1e_1e1​. However, whereas in (7) these get compounded, in MGS these get projected which leads to better numerical properties.

In Julia, it's easy to work in lower precision than 64 bits which can help highlight the benefit. Modifying the code above for the modified variant is trivial, we just need to replace the line R[i, k] = dot(A[:, k], Q[:, i]) for R[i, k] = dot(w, Q[:, i]):

function mgs(A::Matrix{T}; tol=1e-10) where T
    n, m = size(A)
    Q = zeros(T, n, n)
    R = zeros(T, n, m)
    c = 1
    for k = 1:m
        w = A[:, k]
        for i = 1:c-1
            R[i, k] = dot(w, Q[:, i])           # the modification
            w .-= R[i, k] .* Q[:, i]            
        end
        η = norm(w)
        η < tol && continue
        Q[:, c] .= w ./ η
        R[c, k] = dot(A[:, k], Q[:, c])
        c += 1
    end
    p = c - 1
    return Q[:, 1:p], R[1:p, :]
end
