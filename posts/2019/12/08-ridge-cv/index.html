Brief recap

The Ridge regression is a simple penalised regression model corresponding to a L2 loss with L2 penalty on the coefficients:

βridge ⁣ ⁣= ⁣ ⁣arg⁡min⁡∥Xβ−y∥22+λ∥β∥22, β_{\text{ridge}} \quad\!\! =\quad\!\! \arg\min \|X\beta - y\|_2^2 + \lambda \|\beta\|_2^2, βridge​=argmin∥Xβ−y∥22​+λ∥β∥22​,

where XXX is n×pn\times pn×p and λ>0\lambda > 0λ>0 is the penalty (hyper)parameter. We'll drop the subscript "ridge" as it's now obvious from the context.

The minimum verifies

(XtX+λI)β=Xty. (X^tX + λ I) β \quad = \quad X^ty. (XtX+λI)β=Xty.

Complexity of solving Ridge

Tall case (n>pn>pn>p)

The computational cost of solving (2) directly is O(np2+p3)O(np^2 + p^3)O(np2+p3):

  • building XtXX^tXXtX is O(np2)O(np^2)O(np2),

  • solving the resulting p×pp\times pp×p system is O(p3)O(p^3)O(p3).

In the case where n>pn > pn>p (tall case), this is asymptotically dominated by the first term: O(np2)O(np^2)O(np2).

Fat case (p>np>np>n)

In the case where p>np > np>n (fat case), it is more efficient to solve a derived system obtained by pre-multiplying (2) by XXX:

(XXt+λI)Xβ=XXty (XX^t + λ I)Xβ \quad = \quad XX^t y (XXt+λI)Xβ=XXty

indeed, in that case XXtXX^tXXt is n×nn\times nn×n which is then smaller than p×pp \times pp×p. It is useful to show a way to solve this explicitly: consider the SVD X=UΣVtX = UΣ V^tX=UΣVt, then XXt=UΣ2UtXX^t = UΣ^2 U^tXXt=UΣ2Ut and the equation (3) becomes

U(Σ2+λI)ΣVtβ=UΣ2Uty U(Σ^2 + λ I) Σ V^t β \quad=\quad U Σ^2U^t yU(Σ2+λI)ΣVtβ=UΣ2Uty

taking advantage from the fact that UUU is orthogonal so that UUt=IUU^t = IUUt=I (same with VVV). This can be further massaged into

β=VΣ(Σ2+λI)−1Uty=XtU(Σ2+λI)−1Uty,\begin{array}{rcl} β &=& V Σ (Σ^2 + \lambda I)^{-1}U^t y \\ &=& X^t U(Σ^2 + \lambda I)^{-1}U^t y,\end{array}β​==​VΣ(Σ2+λI)−1UtyXtU(Σ2+λI)−1Uty,​

using that X=UΣVtX=UΣ V^tX=UΣVt so that V=XtUΣ−1V = X^t U Σ^{-1}V=XtUΣ−1. Overall, the complexity is asymptotically dominated by the construction of XXtXX^tXXt which is O(pn2)O(pn^2)O(pn2) followed by the complexity of computing its SVD which is O(n3)O(n^3)O(n3).

Conclusion

  • tall case: n>pn > pn>p, O(np2)O(np^2)O(np2), dominated by the construction of XtXX^tXXtX,

  • fat case: n<pn < pn<p, O(pn2)O(pn^2)O(pn2), dominated by the construction of XXtXX^tXXt.

Recycling computations when changing λλλ

When tuning the hyperparameter λλλ, we will potentially want to compute βλβ_λβλ​ for a number of different λλλ; in Ridge regression we can ensure that this is done efficiently so that trying out different λλλ is cheap after having computed the first solution.

We will again consider separately the tall and fat case, and will assume that it is reasonable to form either XtXX^tXXtX or XXtXX^tXXt, for each case we will consider its SVD.

Tall case

Let X=UΣVtX = UΣV^tX=UΣVt and consequently XtX=VΣ2VtX^tX = V\Sigma^2V^tXtX=VΣ2Vt then (2) can be written

V(Σ2+λI)Vtβ=Xty V(Σ^2 + λ I)V^t β \quad =\quad X^ty V(Σ2+λI)Vtβ=Xty

and consequently β=V(Σ2+λI)−1Xty\beta = V(Σ^2 + λ I)^{-1}X^tyβ=V(Σ2+λI)−1Xty.

Once the SVD has been formed, we are left, for any λλλ, with computing VDλzVD_λzVDλ​z where z=Xtyz = X^tyz=Xty can be computed once in O(np)O(np)O(np), Dλ=(Σ2+λI)−1D_λ = (Σ^2 + λ I)^{-1}Dλ​=(Σ2+λI)−1 is a diagonal matrix which is computed and applied in O(p)O(p)O(p) and the application of VVV is O(p2)O(p^2)O(p2).

Overall:

OperationCost
initial computation of XtXX^tXXtXO(np2)O(np^2)O(np2)
initial SVD of XtXX^tXXtXO(p3)O(p^3)O(p3)
computation of VDλzVD_λzVDλ​zO(p2)O(p^2)O(p2) per λλλ

in other words, computing the solution for O(n)O(n)O(n) different λ\lambdaλ is only twice as expensive asymptotically as computing the solution for a single one.

Fat case

In the fat case, we already have the equation (5): β=XtU(Σ2+λI)−1Utyβ = X^t U(Σ^2+λI)^{-1}U^tyβ=XtU(Σ2+λI)−1Uty, we get an analogous argument:

OperationDominating Cost
initial computation of XXtXX^tXXtO(pn2)O(pn^2)O(pn2)
initial SVD of XXtXX^tXXtO(n3)O(n^3)O(n3)
initial computation of w=Utyw = U^tyw=UtyO(pn)O(pn)O(pn)
computation of XtUDλwX^t U D_λ wXtUDλ​wO(np)O(np)O(np) per λλλ

in other words, computing the solution for O(n)O(n)O(n) different λ\lambdaλ is only twice as expensive asymptotically as computing the solution for a single one.

Notes:

  • Observe that for both the tall and fat case, paying around twice the cost of the initial computation, allows to compute the solutions for O(n)O(n)O(n) different λλλ.

  • thus far we have not considered the computation of the intercept but it's not hard to do and does not change the complexity analysis.

Simple code

We can easily check all this with code in Julia; for instance, let's consider the fat case:

using LinearAlgebra

basic_ridge_solve(X, y, λ=1) = (X'X + λ*I) \ X'y

X_fat = randn(50, 500)
y_fat = randn(50)

λ = 1

# slow solve requiring the solution of a 500x500 system
β_fat = basic_ridge_solve(X_fat, y_fat, λ);

Now let's check that we can recover it the cheaper way (via the SVD of XXtXX^tXXt)

# SVD of a 50x50 system
F = svd(Symmetric(X_fat * X_fat'))
U, S = F.U, F.S
w_fat = U'y_fat ./ (S .+ λ)
β_fat ≈ X_fat' * (U * w_fat)
true

Now let's change λλλ:

λ′ = 3

# naive route
β_fat′ = basic_ridge_solve(X_fat, y_fat, λ′)

# efficient route
w_fat′ = U'y_fat ./ (S .+ λ′)
β_fat′ ≈ X_fat' * (U * w_fat′)
true

LOOCV trick

In Leave-One-Out CV, for a given λλλ we want to train the model for each of n−1n-1n−1 cases where we consider only n−1n-1n−1 data points and want to report the error on the last point.

Let X(i)X_{(i)}X(i)​ (resp y(i)y_{(i)}y(i)​) be the matrix (resp. vector) with the iiith row removed, and let β(i)β_{(i)}β(i)​ be the Ridge coefficients in that case. We are interested in the predicted error on the dropped point:

ei=xitβ(i)−yi e_i \quad=\quad x_i^tβ_{(i)} - y_iei​=xit​β(i)​−yi​

We know that for a fixed iii, we can compute that error efficiently for many λ\lambdaλ but it does require one initial SVD. That's a bit annoying because we have to do that for every iii, meaning a lot of SVDs to compute; this seems silly, surely we can re-use some computations!

It's well known that we can indeed speed things up. See for instance Rifkin and Lippert (2007) whose formula is used in Sklearn's RidgeCV model, or Golub, Heath and Wahba (1978) for a statistical perspective. The presentation below is different but the gist in terms of the complexity gain is the same.

Tall case

Let us write X(i)=Ω(i)XX_{(i)} = Ω_{(i)} XX(i)​=Ω(i)​X where ΩiΩ_iΩi​ is the identity matrix with the iiith row removed. We then also have y(i)=Ω(i)yy_{(i)}=\Omega_{(i)}yy(i)​=Ω(i)​y. It is not hard to show that Ω(i)tΩ(i)=(I−D(i))Ω_{(i)}^tΩ_{(i)} = (I - D_{(i)})Ω(i)t​Ω(i)​=(I−D(i)​) where D(i)D_{(i)}D(i)​ is an all-zero matrix with a 1 on the iiith diagonal element.

The iiith Ridge solution verifies (X(i)tX(i)+λI)β(i)=X(i)ty(i)(X_{(i)}^tX_{(i)} + λI)β_{(i)} = X_{(i)}^t y_{(i)}(X(i)t​X(i)​+λI)β(i)​=X(i)t​y(i)​. Using the notations introduced in the previous paragraph, we can rewrite this as

((XtX+λI)−XtD(i)X)β(i)=Xt(I−D(i))y ((X^tX + λI) - X^tD_{(i)}X)β_{(i)} \quad=\quad X^t(I-D_{(i)})y ((XtX+λI)−XtD(i)​X)β(i)​=Xt(I−D(i)​)y

If we write H=(XtX+λI)H = (X^tX + λI)H=(XtX+λI), then we know that it can be efficiently inverted assuming we have initially computed the SVD of XtXX^tXXtX. Note also that XtD(i)X=xixitX^tD_{(i)}X = x_i x_i^tXtD(i)​X=xi​xit​.

On the left-hand side of (8) we have therefore a rank-1 perturbation of an invertible matrix. The inverse of such a matrix can be readily expressed using the Sherman-Morrison formula (see the page on matrix inversion lemmas for details):

(H−xixit)−1=H−1+H−1xixitH−11−xitH−1xi. (H - x_ix_i^t)^{-1} \quad=\quad H^{-1} + {H^{-1}x_i x_i^t H^{-1} \over 1 - x_i^t H^{-1} x_i}. (H−xi​xit​)−1=H−1+1−xit​H−1xi​H−1xi​xit​H−1​.

Plugging this in the lhs of (8), we can get an explicit expression for β(i)β_{(i)}β(i)​. But remember that what we really want is eie_iei​ so that we're more interested in xitβ(i)x_i^t β_{(i)}xit​β(i)​:

xitβ(i)=(xitH−1+xitH−1xixitH−11−xitH−1xi)Xt(I−D(i))y. x_i^tβ_{(i)} \quad=\quad \left(x_i^t H^{-1} + {x_i^tH^{-1}x_i x_i^t H^{-1}\over 1-x_i^tH^{-1}x_i} \right)X^t(I-D_{(i)})y. xit​β(i)​=(xit​H−1+1−xit​H−1xi​xit​H−1xi​xit​H−1​)Xt(I−D(i)​)y.

As it turns out, this can be simplified a fair bit. Note that the second factor can be re-written Xty−XtD(i)yX^ty - X^tD_{(i)}yXty−XtD(i)​y but that last term is simply xiyix_iy_ixi​yi​; let also z=Xtyz=X^tyz=Xty which can be pre-computed and let γ=xitH−1xiγ=x_i^tH^{-1}x_iγ=xit​H−1xi​. Then, massaging a bit, we get

xitβ(i)=xitH−1z−γyi1−γ, x_i^tβ_{(i)} \quad=\quad {x_i^t H^{-1}z - γy_i\over 1-γ}, xit​β(i)​=1−γxit​H−1z−γyi​​,

and, correspondingly, ei=(xitH−1z−yi)/(1−γ)e_i = (x_i^tH^{-1}z - y_i) / (1-γ)ei​=(xit​H−1z−yi​)/(1−γ).

To conclude, with H−1=V(Σ2+λI)−1VtH^{-1} = V(Σ^2+λI)^{-1}V^tH−1=V(Σ2+λI)−1Vt, w=Vtzw = V^tzw=Vtz, gi=Vtxig_i=V^tx_igi​=Vtxi​ and g~i=(Σ2+λI)−1gig̃_i = (Σ^2+λI)^{-1}g_ig~​i​=(Σ2+λI)−1gi​, we can rewrite the computation of eie_iei​ as

ei=g~itw−yi1−g~itgi e_i \quad = \quad {g̃_i^t w - y_i \over 1 - g̃_i^tg_i} ei​=1−g~​it​gi​g~​it​w−yi​​

and the cost of computing all the LOO errors is:

OperationDominating complexity
form XtXX^tXXtX and compute its SVDO(np2)O(np^2)O(np2)
pre-compute z=Xtyz=X^tyz=Xty and w=Vtzw=V^tzw=VtzO(pn)O(pn)O(pn)
(∀i∀i∀i) compute gi=Vtxig_i = V^tx_igi​=Vtxi​O(p2)O(p^2)O(p2)
(∀i,λ∀i,λ∀i,λ) compute g~i=(Σ2+λI)−1gig̃_i = (Σ^2+λI)^{-1} g_ig~​i​=(Σ2+λI)−1gi​O(p)O(p)O(p)
(∀i,λ∀i,λ∀i,λ) compute ei=(g~itw−yi)/(1−g~itgi)e_i = (g̃_i^tw - y_i) / (1-g̃_i^tg_i)ei​=(g~​it​w−yi​)/(1−g~​it​gi​)O(p)O(p)O(p)

So overall, the initial setup cost is dominated by O(np2)O(np^2)O(np2) and the subsequent cost is that of computing each gig_igi​: O(np2)O(np^2)O(np2); finally there's a cost O(npκ)O(npκ)O(npκ) where κκκ is the number of λλλ tested.

In other words, modulo constant factors, it costs the same to get all the LOO errors for O(p)O(p)O(p) different λ\lambdaλ than to get a single Ridge solution!

Fat case

When p>np > np>n, we already know that it's beneficial to consider the SVD of XXt=UΣ2UtXX^t = UΣ^2U^tXXt=UΣ2Ut instead of that of XtXX^tXXtX. Here it turns out that just taking (12) and computing gi,g~ig_i, g̃_igi​,g~​i​ and www in terms of UUU and XXX instead of VVV is all we need to do; for this recall that X=UΣVtX = UΣV^tX=UΣVt so that Vt=Σ−1UtXV^t = Σ^{-1}U^tXVt=Σ−1UtX:

OperationDominating complexity
form K=XXtK = XX^tK=XXt and compute its SVDO(pn2)O(pn^2)O(pn2)
pre-compute w=Σ−1UtKyw=Σ^{-1}U^t K yw=Σ−1UtKyO(n2)O(n^2)O(n2)
(∀i∀i∀i) compute gi=Σ−1UtXxig_i = Σ^{-1} U^t X x_igi​=Σ−1UtXxi​O(np)O(np)O(np)
(∀i,λ∀i,λ∀i,λ) compute g~i=(Σ2+λI)−1gig̃_i = (Σ^2+λI)^{-1} g_ig~​i​=(Σ2+λI)−1gi​O(n)O(n)O(n)
(∀i,λ∀i,λ∀i,λ) compute ei=(g~itw−yi)/(1−g~itgi)e_i = (g̃_i^tw - y_i) / (1-g̃_i^tg_i)ei​=(g~​it​w−yi​)/(1−g~​it​gi​)O(n)O(n)O(n)

So overall, the initial setup cost is dominated by O(pn2)O(pn^2)O(pn2) and the subsequent cost is that of computing each gig_igi​: O(pn2)O(pn^2)O(pn2); finally there's a cost O(n2κ)O(n^2κ)O(n2κ) where κκκ is the number of λλλ tested.

Again, modulo constant factors, it costs the same to get all the LOO errors for O(p)O(p)O(p) different λλλ than to get a single Ridge solution.

Simple code

We can easily check all this with code again, let's extend the previous case:

using InvertedIndices
i = 5   # random index between 1 and n
λ = 2.5

X̃ᵢ = X_fat[Not(i),:]
ỹᵢ = y_fat[Not(i),:]
β̃ᵢ = basic_ridge_solve(X̃ᵢ, ỹᵢ, λ)

xᵢ = vec(X_fat[i,:])
yᵢ = y_fat[i]
rᵢ = dot(xᵢ, β̃ᵢ) - yᵢ; # this the i-th LOO residual

Now let's show we can recover this using the previous point:

K = Symmetric(X_fat * X_fat')
F = svd(K)
U, Ut, S = F.V, F.Vt, F.S
Σ² = S
Σ  = sqrt.(S)

w  = (Ut * (K * y_fat)) ./ Σ
gᵢ = (Ut * (X_fat * xᵢ)) ./ Σ
g̃ᵢ = gᵢ ./ (Σ² .+ λ)

rᵢ ≈ (dot(g̃ᵢ, w) - yᵢ) / (1 - dot(g̃ᵢ, gᵢ))
true

Now if we change iii and λλλ,

i = 7
λ = 5.5

X̃ᵢ = X_fat[Not(i),:]
ỹᵢ = y_fat[Not(i),:]
β̃ᵢ = basic_ridge_solve(X̃ᵢ, ỹᵢ, λ)

xᵢ = vec(X_fat[i,:])
yᵢ = y_fat[i]
rᵢ = dot(xᵢ, β̃ᵢ) - yᵢ;

we only have to recompute gᵢ and g̃ᵢ:

gᵢ = (Ut * (X_fat * xᵢ)) ./ Σ
g̃ᵢ = gᵢ ./ (Σ² .+ λ)

rᵢ ≈ (dot(g̃ᵢ, w) - yᵢ) / (1 - dot(g̃ᵢ, gᵢ))
true

Conclusion

In short, provided that computing the SVD of either a p×pp \times pp×p or n×nn\times nn×n matrix is manageable, it's not much more expensive to compute a single Ridge solution than to compute a bunch of solutions or to compute all the LOO errors for a number of different λλλ.

This LOO trick can be generalised to an extent to all leave-some-out schemes; this is covered in the second part.
