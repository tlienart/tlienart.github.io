GMRES and Krylov subspaces

Usually in the literature people present Krylov subspaces first and GMRES and related techniques after. Here we do the opposite as we came to GMRES in a different way. Starting from the iterative procedure, we now connect the dots with the Krylov subspaces.

For chronology's sake, note that Krylov subspaces where introduced by A. Krylov before the 1950s while GMRES was discussed by Y. Saad and M. Schultz in 1986 (Saad and Schultz (1986)).

Back to the present, we discussed iterative methods for solving Ax=bAx=bAx=b with AAA non singular and AAA and bbb with real entries:

xk ⁣ ⁣= ⁣ ⁣xk−1+αkpk ⁣ ⁣= ⁣ ⁣x0+∑i=1kαipi, x_k \quad\!\! =\quad\!\! x_{k-1} + \alpha_k p_k \quad\!\! =\quad\!\! x_0 + \sum_{i=1}^k \alpha_i p_i , xk​=xk−1​+αk​pk​=x0​+i=1∑k​αi​pi​,

for some linearly independent pip_ipi​ and coefficients αi\alpha_iαi​. One of the choice of directions is pk=rk−1p_k=r_{k-1}pk​=rk−1​, which corresponds to the classical GMRES algorithm.

Observe that, for this choice, we have xk=xk−1+αkrk−1x_k=x_{k-1}+\alpha_kr_{k-1}xk​=xk−1​+αk​rk−1​ and

rk ⁣ ⁣= ⁣ ⁣Axk−b=rk−1+αkArk−1=(I+αkA)rk−1=(I+αkA)(I+αk−1A)…(I+α1A)r0\begin{array}{rcl} r_k \quad\!\! =\quad\!\! Ax_k - b &=& r_{k-1} + \alpha_k Ar_{k-1} \\ &=& (I + \alpha_kA) r_{k-1} \\ &=& (I + \alpha_kA)(I +\alpha_{k-1}A)\dots(I+\alpha_1A)r_0\end{array}rk​=Axk​−b​===​rk−1​+αk​Ark−1​(I+αk​A)rk−1​(I+αk​A)(I+αk−1​A)…(I+α1​A)r0​​

Expanding that product, we could find some γ1,…,γk\gamma_1,\dots,\gamma_kγ1​,…,γk​ such that

rk ⁣ ⁣= ⁣ ⁣r0+∑i=1kγiAir0. r_k \quad\!\! =\quad\!\! r_0 + \sum_{i=1}^k \gamma_iA^ir_0. rk​=r0​+i=1∑k​γi​Air0​.

The coefficients γi\gamma_iγi​ don't matter, what does matter is that this is a linear combination of the vectors Air0A^{i}r_0Air0​ for i=0,…,ki=0,\dots,ki=0,…,k which span the Krylov subspace Kk+1\mathcal K_{k+1}Kk+1​ with

Kk(A,r0) ⁣ ⁣= ⁣ ⁣span{r0,Ar0,…,Ak−1r0}, \mathcal K_k(A, r_0) \quad\!\! =\quad\!\! \mathrm{span}\{r_0, Ar_0, \dots, A^{k-1}r_0\}, Kk​(A,r0​)=span{r0​,Ar0​,…,Ak−1r0​},

we will just write Kk\mathcal K_kKk​ when AAA and r0r_0r0​ are obvious from the context.

Since the rk−1r_{k-1}rk−1​ and therefore the pkp_kpk​ each live in Kk\mathcal K_kKk​, the xkx_kxk​ given by (1) are also in Kk\mathcal K_kKk​. The iteration can then be interpreted as iteratively attempting to represent xxx into successive Krylov subspaces.

Standard GMRES corresponds to the choice pk=rkp_k=r_kpk​=rk​ with the minimum residual criterion (cf. previous post). Overall, this can now be re-expressed as:

xk+1−x0 ⁣ ⁣= ⁣ ⁣arg⁡min⁡z∈Kk ∥A(x0+z)−b∥2. x_{k+1} - x_0 \quad\!\! =\quad\!\! \arg \min_{z \in \mathcal K_{k}}\,\, \| A(x_0+z) - b \|_2. xk+1​−x0​=argz∈Kk​min​∥A(x0​+z)−b∥2​.

Petrov-Galerkin condition and FOM

In the previous post, we mostly discussed the choice of directions pkp_kpk​ and not the criterion for choosing the approximation xkx_kxk​ which we fixed at the minimum residual criterion.

A different criterion that can be considered is the Petrov-Galerkin condition requiring the residual rkr_krk​ to be orthogonal to the space spanned by the directions {p1,…,pk}\{p_1, \dots, p_k\}{p1​,…,pk​}. This condition leads to the Full Orthogonalisation Method or FOM Saad (2003).

For generic matrices, this method does not perform very well, but for specific matrices, it leads to the very effective Conjugate Gradient method as will be seen in the next point.

If we denote {q1,…,qk}\{q_1,\dots,q_k\}{q1​,…,qk​} the orthonormal basis corresponding to the pkp_kpk​, the Petrov-Galerkin condition amounts to ⟨rk,qi⟩=0\left\langle r_k, q_i\right\rangle=0⟨rk​,qi​⟩=0 for i=1,…,ki=1,\dots,ki=1,…,k which, in matrix form, reads

(Q(k))trk ⁣ ⁣= ⁣ ⁣0. (Q^{(k)})^tr_k \quad\!\! =\quad\!\! 0. (Q(k))trk​=0.

where Q(k)Q^{(k)}Q(k) is the n×kn\times kn×k matrix with columns {q1,…,qk}\{q_1,\dots,q_k\}{q1​,…,qk​}. Following the same approach as in the discussion of the minres criterion, we can iteratively form Q~(k)\tilde{Q}^{(k)}Q~​(k) such that AQ(k)=Q~(k)R(k)AQ^{(k)} = \tilde{Q}^{(k)}R^{(k)}AQ(k)=Q~​(k)R(k). With xk=x0+∑i=1k−1βiAqix_k = x_0 + \sum_{i=1}^{k-1}\beta_i A q_ixk​=x0​+∑i=1k−1​βi​Aqi​ for some βi\beta_iβi​ to determine and letting β(k)\beta^{(k)}β(k) the vector with components {β1,…,βk}\{\beta_1,\dots,\beta_k\}{β1​,…,βk​}, the condition then reads

(Q(k))tQ~(k)R(k)β(k) ⁣ ⁣= ⁣ ⁣−(Q(k))tr0. (Q^{(k)})^t\tilde{Q}^{(k)}R^{(k)}\beta^{(k)} \quad\!\! =\quad\!\! -(Q^{(k)})^tr_0. (Q(k))tQ~​(k)R(k)β(k)=−(Q(k))tr0​.

Like in the minres case, this is just a regression problem in Rk\mathbb R^kRk and some tricks can be applied to solve this cheaply. We dive much deeper though as, in the general case, FOM behaves much worse than GMRES as is illustrated below in a rough benchmark.

(Rough) implementation

Most of the code here is drawn from the previous post, we simplify the resolution of the least-square problem to get the β\betaβ using the generic backslash operator (we shouldn't but it makes the code simpler). The main goal here is to get a quick comparison of using the minres vs the Petrov-Galerkin criterion.

using LinearAlgebra, StableRNGs

function mgs!(Q::Matrix, R::Matrix, v::Vector, k::Int)
    n = length(v)
    Q[:, k] .= v
    for i = 1:k-1
        R[i, k] = dot(Q[:, k], Q[:, i])
        Q[:, k] .-= R[i, k] .* Q[:, i]
    end
    Q[:, k] = normalize(Q[:,k])
    R[k, k] = dot(v, Q[:, k])
    return
end

function itersolve(A::Matrix, b::Vector, x0::Vector;
                   niter::Int=10, dirs=:rand, criterion=:minres)
    start = time()
    n = size(A, 1)
    r0 = A * x0 - b

    xk = copy(x0)
    rk = copy(r0)
    pk = zeros(n)
    Q = zeros(n, niter)
    Q̃ = zeros(n, niter)
    R = zeros(niter, niter)
    M = zeros(niter, niter)
    norm_rk = zeros(niter + 1)
    norm_rk[1] = norm(r0)
    times = zeros(niter+1)
    times[1] = time() - start

   #@views for k = 1:niter
   for k = 1:niter
       if dirs == :rand
           pk = randn(n)
       elseif dirs == :krylov
           pk = rk
       elseif dirs == :grad
           pk = A' * rk
       end
       mgs!(Q, R, pk, k)
       mgs!(Q̃, R, A*Q[:, k], k)

       if criterion == :minres
           γk = -Q̃[:, 1:k]' * r0
           βk = R[1:k, 1:k] \ γk
       elseif criterion == :fom
           # compute M = (Q'Q̃) iteratively
           if k > 1
               M[1:k-1, k] .= Q[:, 1:k-1]' * Q̃[:, k]
               M[k, 1:k-1] .= vec(Q[:, k]' * Q̃[:, 1:k-1])
           end
           M[k, k] = dot(Q[:, k], Q̃[:, k])
           # solve (Q'Q̃R)β ≈ (-Q'r0)
           βk = (M[1:k, 1:k] * R[1:k, 1:k]) \ (-Q[:, 1:k]' * r0)
       end
       xk .= x0 .+ Q[:, 1:k] * βk
       rk .= A * xk .- b
       norm_rk[k+1] = norm(rk)
       times[k+1] = time() - start
   end
   return xk, norm_rk, times
end
itersolve (generic function with 1 method)

As usual, the code above is not optimised, particularly at places S1 and S2, a dedicated procedure should be used rather than the generic backslash operator.

Our way to compute the coefficients corresponding to the FOM criterion is very crude but our goal here is simply to have a quick comparison with the other.

As can be seen on the graph above, FOM can have a much less smooth convergence and GMRES is usually preferred to it (at least to the naive form presented above).

Conjugate gradient method

While FOM typically underperforms GMRES-style updates, in the case where AAA is symmetric positive definite (SPD) it can be modified to lead to a very useful and efficient algorithm: the conjugate gradient method.

For the rest of this section, AAA is assumed to be SPD i.e. ⟨x,Ax⟩=⟨Ax,x⟩>0\left\langle x, Ax\right\rangle = \left\langle Ax, x\right\rangle > 0⟨x,Ax⟩=⟨Ax,x⟩>0 for any x≠0x\neq 0x=0. Recall that if AAA is SPD it induces an inner-product on Rn\mathbb R^nRn with

⟨x,y⟩A ⁣ ⁣= ⁣ ⁣⟨x,Ay⟩. \left\langle x, y\right\rangle_A \quad\!\! =\quad\!\! \left\langle x, Ay\right\rangle. ⟨x,y⟩A​=⟨x,Ay⟩.

Indeed, it is trivially bilinear, symmetric since ⟨x,y⟩A=⟨x,Ay⟩=⟨Ax,y⟩=⟨y,x⟩A\left\langle x, y\right\rangle_A = \left\langle x, Ay\right\rangle = \left\langle Ax, y\right\rangle = \left\langle y, x\right\rangle_A⟨x,y⟩A​=⟨x,Ay⟩=⟨Ax,y⟩=⟨y,x⟩A​ and positive definite since AAA is positive definite. Correspondingly, we can define a norm ∥x∥A2=⟨x,Ax⟩\|x\|_A^2 = \left\langle x, Ax\right\rangle∥x∥A2​=⟨x,Ax⟩.

Petrov-Galerkin condition again

Let's start with the basic iteration again, assuming we have an iteratively constructed set of linearly-independent {p1,…,pk}\{p_1,\dots,p_k\}{p1​,…,pk​} which we express in some base {u1,…,uk}\{u_1,\dots,u_k\}{u1​,…,uk​}. Then, the iteration amount to

{xk=x0+∑i=1kαkukrk=r0+∑i=1kαkAuk \begin{cases} x_k &=& x_0 + \sum_{i=1}^k \alpha_k u_k \\ r_k &=& r_0 + \sum_{i=1}^k \alpha_k Au_k \end{cases} {xk​rk​​==​x0​+∑i=1k​αk​uk​r0​+∑i=1k​αk​Auk​​

with the αi\alpha_iαi​ such that rkr_krk​ is perpendicular to the space spanned by the pkp_kpk​ (and so the uku_kuk​) i.e.:

⟨rk,uj⟩ ⁣ ⁣= ⁣ ⁣0forj=1,…,k. \left\langle r_k, u_j\right\rangle \quad\!\! =\quad\!\! 0 \quad \text{for}\quad j=1,\dots,k. ⟨rk​,uj​⟩=0forj=1,…,k.

Now since rk=rk−1+αkAukr_k = r_{k-1} + \alpha_k Au_krk​=rk−1​+αk​Auk​, and that, by construction, rk−1r_{k-1}rk−1​ must be orthogonal to uju_juj​ for 1,…,k−11,\dots,k-11,…,k−1, the condition becomes

⟨Auk,uj⟩=0,forj=1,…,k−1⟨rk−1,uk⟩+αk⟨Auk,uk⟩=0\begin{array}{rcl} \left\langle Au_k, u_j\right\rangle &=& 0, \quad\text{for}\quad j=1,\dots,k-1 \\ \left\langle r_{k-1}, u_k\right\rangle + \alpha_k\left\langle Au_k, u_k\right\rangle &=& 0 \end{array}⟨Auk​,uj​⟩⟨rk−1​,uk​⟩+αk​⟨Auk​,uk​⟩​==​0,forj=1,…,k−10​

In the case where AAA is SPD, we can construct a base {u1,…,uk}\{u_1,\dots,u_k\}{u1​,…,uk​} to make the above condition particularly simple to verify. Before doing that, we give an alternative interpretation of using the PG condition.

Correspondance between Petrov-Galerkin and AAA-projections

With the choice pk=rk−1p_k=r_{k-1}pk​=rk−1​, the {p1,…,pk}\{p_1,\dots,p_k\}{p1​,…,pk​} span the Krylov subspace Kk(A,r0)\mathcal K_k(A, r_0)Kk​(A,r0​), and the orthogonality condition requires that rkr_krk​ be orthogonal to Kk\mathcal K_kKk​. When AAA is SPD, we show below that the iterations with the PG condition are equivalent to the following iterations:

xk−x0=arg⁡min⁡z∈Kk  ∥x‾−z∥A2\begin{array}{rcl} x_{k} - x_0 &=& \arg\displaystyle{\min_{z\in\mathcal K_k}} \,\, \|\overline{x}-z\|_A^2\\\end{array}xk​−x0​​=​argz∈Kk​min​∥x−z∥A2​​

where x‾=x−x0\overline{x} = x-x_0x=x−x0​ and ∥u∥A2=⟨u,Au⟩\|u\|^2_A=\left\langle u,Au\right\rangle∥u∥A2​=⟨u,Au⟩. In other words, a sequence of projections of (x−x0)(x-x_0)(x−x0​) onto Kk\mathcal K_kKk​ in the geometry induced by AAA ("AAA-projections").

In order to show the equivalence, let's expand the AAA-norm and drop the term that doesn't depend on zzz:

arg⁡min⁡z∈Kk  ⟨z,Az⟩−⟨x‾,Az⟩. \arg\min_{z \in \mathcal K_k} \,\, \left\langle z, Az\right\rangle - \left\langle \overline{x}, Az\right\rangle. argz∈Kk​min​⟨z,Az⟩−⟨x,Az⟩.

This is a convex minimisation problem since the objective is convex (as AAA is SPD) over a convex set Kk\mathcal K_kKk​ (convex since it's an affine subspace). We can thus consider the first order optimality condition (FOC) of the problem. The gradient of the objective function is 2A(z−x‾)2A(z-\overline{x})2A(z−x) and the FOC at the minimiser z†z^\daggerz† is:

−2A(z†−x‾) ⁣ ⁣∈ ⁣ ⁣{y∈Rn  ∣ ⟨w−z†,y⟩≤0,  ∀w∈Kk} -2A(z^\dagger -\overline{x}) \quad\!\! \in\quad\!\! \{ y \in\mathbb R^n\,\, |\, \left\langle w - z^\dagger, y\right\rangle \le 0, \,\, \forall w\in\mathcal K_k \} −2A(z†−x)∈{y∈Rn∣⟨w−z†,y⟩≤0,∀w∈Kk​}

where the left-hand-side is the normal cone to Kk\mathcal K_kKk​. This amounts to requiring

⟨w−z†,A(x‾−z)⟩ ⁣ ⁣≤ ⁣ ⁣0,   ∀w∈Kk. \left\langle w - z^\dagger, A(\overline{x} - z)\right\rangle \quad\!\! \le\quad\!\! 0, \,\,\, \forall w \in \mathcal K_k. ⟨w−z†,A(x−z)⟩≤0,∀w∈Kk​.

Since both www and z†z^\daggerz† are in Kk\mathcal K_kKk​, we can let u=w−z†u=w-z^\daggeru=w−z† and require ⟨x‾−z†,Au⟩≤0\left\langle \overline{x}-z^\dagger, Au\right\rangle \le 0⟨x−z†,Au⟩≤0 for any u∈Kku \in \mathcal K_ku∈Kk​. For this to always hold we must have (x‾−z†)(\overline{x}-z^\dagger)(x−z†) be AAA-orthogonal to Kk\mathcal K_kKk​ or:

⟨x‾−z†,Au⟩ ⁣ ⁣= ⁣ ⁣0   ∀u∈Kk. \left\langle \overline{x} - z^\dagger, Au\right\rangle \quad\!\! =\quad\!\! 0\,\,\,\forall u\in\mathcal K_k. ⟨x−z†,Au⟩=0∀u∈Kk​.

Equivalently, ⟨Ax‾−Az†,u⟩=0\left\langle A\overline{x}-Az^\dagger, u\right\rangle=0⟨Ax−Az†,u⟩=0 for all u∈Kku \in \mathcal K_ku∈Kk​ or

⟨Ax−A(x0+z†),u⟩ ⁣ ⁣= ⁣ ⁣0   ∀u∈Kk. \left\langle Ax - A(x_0+z^\dagger), u\right\rangle \quad\!\! =\quad\!\! 0 \,\,\,\forall u\in\mathcal K_k. ⟨Ax−A(x0​+z†),u⟩=0∀u∈Kk​.

with xk=x0+z†x_{k}=x_0+z^\daggerxk​=x0​+z†, Ax=bAx=bAx=b and rk=Axk−br_k=Ax_k-brk​=Axk​−b, the above equation is equivalent to requiring rkr_krk​ to be perpendicular to Kk\mathcal K_kKk​ which is the Petrov-Galerkin condition.

Note if you're not familiar with the FOC with the normal cone, you can consider this post. Another way to put the condition (which you might have seen in the context of the KKT conditions) is that, at a minimiser of a convex constrained minimisation problem, either the gradient is zero at that point or the gradient is perpendicular to the boundary of the domain at that point.

AAA-orthonormal basis

Let us introduce the notation ∝A\propto_A∝A​, similar to our previous ∝\propto∝ with x∝Ayx\propto_A yx∝A​y meaning x=y/∥y∥Ax = y/\|y\|_Ax=y/∥y∥A​ for a nonzero yyy. Then, in much the same way that we can use Gram-Schmidt to produce an orthonormal basis {q1,…,qk}\{q_1,\dots,q_k\}{q1​,…,qk​} out of a set of linearly independent vectors {p1,…,pk}\{p_1, \dots, p_k\}{p1​,…,pk​} with

qk ⁣ ⁣∝ ⁣ ⁣pk−∑i=1k−1⟨pk,qi⟩qi q_k \quad\!\! \propto\quad\!\! p_k - \sum_{i=1}^{k-1} \left\langle p_k, q_i\right\rangle q_i qk​∝pk​−i=1∑k−1​⟨pk​,qi​⟩qi​

and ⟨qi,qj⟩=δij\left\langle q_i,q_j\right\rangle = \delta_{ij}⟨qi​,qj​⟩=δij​, we can produce an AAA-orthonormal basis {u1,…,uk}\{u_1,\dots,u_k\}{u1​,…,uk​} with

uk ⁣ ⁣∝A ⁣ ⁣pk−∑i=1k−1⟨pk,qi⟩Aqi u_k \quad\!\! \propto_A\quad\!\! p_k - \sum_{i=1}^{k-1} \left\langle p_k, q_i\right\rangle_A q_i uk​∝A​pk​−i=1∑k−1​⟨pk​,qi​⟩A​qi​

and ⟨ui,uj⟩A=δij\left\langle u_i, u_j\right\rangle_A = \delta_{ij}⟨ui​,uj​⟩A​=δij​.

Working in a AAA-orthonormal basis

If we have iteratively constructed an AAA-orthonormal basis {u1,…,uk}\{u_1,\dots,u_k\}{u1​,…,uk​}, then verifying the condition (11) simply requires

αk ⁣ ⁣= ⁣ ⁣−⟨rk−1,uk⟩. \alpha_k \quad\!\! =\quad\!\! -\left\langle r_{k-1}, u_k\right\rangle. αk​=−⟨rk−1​,uk​⟩.

This is already fairly nice. Even nicer though is that constructing the AAA-orthonormal basis with the "Krylov" direction choice (i.e. pk=rk−1p_k=r_{k-1}pk​=rk−1​) can be done with a simple recurrence as shown below.

The Gram-Schmidt step to obtain uk+1u_{k+1}uk+1​ when pk+1=rkp_{k+1}=r_{k}pk+1​=rk​ is

Implementation

In the simple implementation below, we use three pairs (u,Au)(u, Au)(u,Au) of vectors in Rn\mathbb R^nRn for storage as well as xxx and rrr (we could even reduce that). The important note is that the recurrence relation allows to compute the vectors uku_kuk​ iteratively without having to keep all of them in memory (unlike GMRES-style iterations which have linearly-growing memory requirements).

function conjgrad(A::Matrix, b::Vector, x0::Vector; niter=10)
    x = copy(x0)
    r = A * x .- b

    norm_rk    = zeros(niter+1)
    norm_rk[1] = norm(r)

    # We need to compute the first two steps
    # before starting to use the recurrence

    u1   = r
    Au1  = A * u1
    η1   = sqrt(dot(u1, Au1))
    u1  /= η1
    Au1 /= η1
    α    = -dot(r, u1)
    x  .+= α .* u1
    r  .+= α .* Au1

    norm_rk[2] = norm(r)

    u2  = r .- dot(r, Au1) .* u1
    Au2 = A * u2
    η2  = sqrt(dot(u2, Au2))
    u2  /= η2
    Au2 /= η2
    α    = -dot(r, u2)
    x  .+= α .* u2
    r  .+= α .* Au2

    norm_rk[3] = norm(r)

    u  = copy(u2)
    Au = copy(Au2)

    for k = 1:niter-2
        # Recurrence relation
        u   .= -(α * dot(Au2, Au1)) .* u1 .+ (η2 - dot(r, Au2)) .* u2 .+ (α .* Au2)
        Au  .= A * u
        η    = sqrt(dot(u, Au))
        u  ./= η
        Au ./= η

        α   = -dot(r, u)
        x .+= α .* u
        r .+= α .* Au

        η2    = η
        u1   .= u2
        Au1  .= Au2
        u2   .= u
        Au2  .= Au

        norm_rk[3+k] = norm(r)
    end
    return x, norm_rk
end

In the implementation above, the step with dominant complexity is the matrix-vector multiplication to compute AuAuAu, everything else has negligible complexity in comparison.

Comparison

In the example below, we construct a simple and well-conditioned SPD system, and compare the various methods discussed so far. Again, this is not meant as a thorough benchmark but rather as a way to make a couple of interesting observations.

using IterativeSolvers

begin
    rng = StableRNG(111)

    # construct a simple SPD matrix
    n = 100
    A = randn(rng, n, n)
    A = A'A + 2I

    κA = round(cond(A), sigdigits=3)
    println("Condition number κ(A): $κA")

    x = randn(rng, n)
    b = A * x
    x0 = randn(rng, n)

    xcg, nrk = conjgrad(A, b, copy(x0); niter=n)
    res_krylov = itersolve(A, b, x0; niter=n, dirs=:krylov, criterion=:minres)
    res_grad = itersolve(A, b, x0; niter=n, dirs=:grad, criterion=:minres)
    res_fom_k = itersolve(A, b, x0; niter=n, dirs=:krylov, criterion=:fom)
    res_fom_g = itersolve(A, b, x0; niter=n, dirs=:grad, criterion=:fom)

    xcg, lcg = cg!(copy(x0), A, b; maxiter=n, log=true, reltol=0)
    ncg = [norm(A*x0-b), lcg.data[:resnorm]...]

    figure(figsize=(8, 6))
    semilogy(nrk, label="CG", marker="v")
    semilogy(res_krylov[2], label="GMRES (krylov)")
    semilogy(res_grad[2], label="GMRES (grad)")
    semilogy(res_fom_k[2], label="FOM (krylov)", linestyle="--")
    semilogy(res_fom_g[2], label="FOM (grad)", linestyle="--")
    semilogy(ncg, label="CG (IterativeSolvers)", marker="x")
    xlabel("Number of iterations")
    ylabel("Norm of the residuals")
    legend()
end
Condition number κ(A): 194.0

On the graph above we can see a few things:

  • Our implementation of CG gives essentially the same results as that of the IterativeSolvers package which gives some confidence in the implementation of conjgrad in terms of correctness,

  • GMRES and FOM with Krylov directions have a very similar behaviour and are better than CG for a large number of iterations,

  • GMRES and FOM with "Grad" directions have a very similar behaviour and are quite a bit worse than the other two.

Recall that all these methods require the same complexity per step (dominated by one matrix-vector product) with the exception of the ones using the grad direction (two matrix-vector products).

The first point above is the most important one at this point, the other two help highlight that it can be difficult to compare iterative methods in general. Consider for instance a very similar example but with much worse condition number:

What we didn't discuss

In these two posts, the main goal was to re-obtain GMRES and CG from scratch and in a constructive fashion. There's however quite a few interesting things that we didn't discuss to try to keep the presentation not too cluttered, these include:

  • how these iterative methods relate to the Hessenberg decomposition where A=Q′HQA=Q'HQA=Q′HQ for an orthonormal matrix QQQ and a Hessenberg matrix HHH (a matrix with hij=0h_{ij}=0hij​=0 for i>j+1i > j+1i>j+1),

  • how the iterative orthogonalisation procedure (Modified Gram-Schmidt) in the case of Krylov subspaces connect to methods for obtaining eigenvalues (Arnoldi iteration and Lanczos algorithm in the symmetric case) (see e.g. Strang (2006)),

  • how these iterative methods can suffer from stalling and how to deal with it, and what kind of guarantees they can offer,

  • the use of preconditioning to speed up convergence,

  • extensions such as BiCGStab van der Vorst (1992).
